## 1. 项目概述

### 1.1 项目定位

**墨韵·智库**是一个面向年轻创作者、技术学习者、文学爱好者的**文学+技术双内容形态共存的社区平台**。项目采用现代化技术栈，构建完整的内容生产、分发推荐、用户成长、商业化体系。

### 1.2 核心价值

- 为创作者提供优质的内容发布和展示平台
- 为读者提供丰富的内容资源和良好的阅读体验
- 构建活跃的社区氛围，促进用户互动
- 探索内容平台的商业化模式

### 1.3 项目特色

| 特色 | 说明 |
|------|------|
| **双形态内容** | 技术流和文学派两种内容展示风格 |
| **独立认证** | 前后台用户完全独立，双 SecurityFilterChain 配置 |
| **多主题支持** | 日间、夜间、护眼三种主题自由切换 |
| **响应式设计** | 完美适配各种屏幕尺寸 |
| **工作流引擎** | Flowable 集成，支持复杂的业务审批流程 |
| **现代技术栈** | Vue 3 + Spring Boot 3 + MyBatis-Plus |

## 2. 技术架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      客户端层                            │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────┐      ┌──────────────────────────┐  │
│  │   moyun-portal  │      │     moyun-admin-vue     │  │

│  │  Vue 3 + Vite   │      │   Vue 3 + Element Plus   │  │
│  └─────────────────┘      └──────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                           ↓ HTTP/REST
┌─────────────────────────────────────────────────────────┐
│                      服务端层                            │
├─────────────────────────────────────────────────────────┤
│              moyun-server (Spring Boot 3)               │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │  Portal API │  │   CMS API   │  │   System API    │  │
│  │  /portal/*  │  │   /cms/*    │  │   /system/*     │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
│  ┌─────────────────────────────────────────────────────┐│
│  │              核心框架层                              ││
│  │  Security | MyBatis-Plus | Redis | Flowable       ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                      数据层                              │
├─────────────────────────────────────────────────────────┤
│  MySQL 8.0+              Redis              文件存储    │
```

## 6. 实施建议

### 6.1 第三阶段实施建议

#### 系统配置管理

**建议实现方式**:
1. 创建 `sys_config` 表存储配置
2. 开发配置管理 CRUD 接口
3. 前端页面使用 Key-Value 形式展示配置
4. 配置项分类：站点信息、SEO参数、邮件配置、短信配置

**快速开发路径**:
```
Day 1: 数据库设计 + 后端CRUD
Day 2: 前端页面开发
Day 3: 联调测试 + 部署
```

#### 数据统计分析

**建议实现方式**:
1. 使用 MySQL 统计函数
2. Redis 缓存统计结果
3. 前端使用 ECharts 展示图表
4. 定时任务更新统计数据

**快速开发路径**:
```
Day 1: 统计接口开发
Day 2: 可视化页面开发
Day 3: 仪表盘整合
```

#### SEO基础优化

**建议实现方式**:
1. 后端添加动态 Meta 标签生成
2. 前端使用 @vueuse/head
3. 生成 sitemap.xml
4. 配置 robots.txt

**快速开发路径**:
```
Day 1: Meta标签优化
Day 2: sitemap生成
Day 3: 结构化数据
```
