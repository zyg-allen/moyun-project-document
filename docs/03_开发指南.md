## 二、代码规范

### 2.1 Java代码规范

#### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | 大驼峰 | `UserController` |
| 方法名 | 小驼峰 | `getUserInfo()` |
| 变量名 | 小驼峰 | `userName` |
| 常量名 | 全大写+下划线 | `MAX_SIZE` |
| 包名 | 全小写 | `com.moyun.modules.system` |

#### 文件组织

```
modules/system/
├── controller/     # REST控制器
├── service/        # 业务层（接口+实现）
├── mapper/         # 数据访问层
├── domain/         # 领域模型
│   ├── entity/     # 实体类
│   ├── dto/        # 数据传输对象
│   ├── vo/         # 视图对象
│   └── query/      # 查询对象
└── config/         # 配置类（如有）
```

#### 代码风格

- 使用 Lombok 简化代码（`@Data`, `@Builder`）
- 添加 Swagger 注解（`@Api`, `@ApiOperation`）
- 添加 JSR-303 验证注解（`@NotBlank`, `@Size`）
- 编写完整的 Javadoc 注释

## 三、开发流程

### 3.1 新增业务模块流程

1. **创建目录结构**
```bash
mkdir -p modules/your-module/{controller,service,mapper,domain/{entity,dto,vo,query}}
```

2. **创建实体类**
```java
@Data
@TableName("your_table")
public class YourEntity extends BaseEntity {
    private String name;
    // ...
}
```

3. **创建DTO**
```java
@Data
public class CreateYourDTO {
    @NotBlank(message = "名称不能为空")
    private String name;
}
```

4. **创建VO**
```java
@Data
public class YourVO {
    private Long id;
    private String name;
}
```

5. **创建Mapper**
```java
public interface YourMapper extends BaseMapper<YourEntity> {
    Page<YourEntity> selectPage(Page<YourEntity> page, @Param("query") YourQuery query);
}
```

6. **创建Service**
```java
public interface IYourService {
    Page<YourVO> selectPage(YourQuery query);
}

@Service
public class YourServiceImpl extends ServiceImpl<YourMapper, YourEntity> implements IYourService {
    @Override
    public Page<YourVO> selectPage(YourQuery query) {
        Page<YourEntity> page = PageUtils.buildPage(query);
        page = baseMapper.selectPage(page, query);
        return page.convert(entity -> BeanUtil.copyProperties(entity, YourVO.class));
    }
}
```

7. **创建Controller**
```java
@RestController
@RequestMapping("/your-module")
public class YourController extends BaseController {
    @Autowired
    private IYourService yourService;
    
    @GetMapping("/list")
    public TableDataInfo list(YourQuery query) {
        Page<YourVO> page = yourService.selectPage(query);
        return getDataTable(page);
    }
}
```

### 3.2 新增前端页面流程

1. **创建API文件**
```typescript
// src/api/yourModule.ts
export const getList = (params: YourQuery) => {
  return httpGetList('/your-module/list', params);
};
```

2. **创建页面组件**
```vue
<template>
  <div class="your-page">
    <!-- 页面内容 -->
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { getList } from '@/api/yourModule';

const list = ref([]);

onMounted(async () => {
  const res = await getList({});
  list.value = res.list;
});
</script>
```

3. **配置路由**
```typescript
// src/router/index.ts
{
  path: '/your-page',
  name: 'YourPage',
  component: () => import('@/pages/YourPage.vue'),
}
```

## 四、最佳实践

### 4.1 后端最佳实践

#### 异常处理

- 使用全局异常处理器 `GlobalExceptionHandler`
- 自定义业务异常 `BusinessException`
- 统一返回格式 `AjaxResult`

```java
// Service层抛出业务异常
throw new BusinessException("业务处理失败");

// Controller层返回统一格式
return AjaxResult.success(data);
return AjaxResult.error("操作失败");
```

#### 事务管理

- 使用 `@Transactional` 注解
- 合理设置事务传播级别
- 避免在循环中调用事务方法

```java
@Transactional(rollbackFor = Exception.class)
public void batchSave(List<YourEntity> list) {
    list.forEach(this::save);
}
```

#### 分页查询

- 使用 MyBatis-Plus 分页插件
- 统一使用 `PageUtils.buildPage(query)` 构建分页对象
- 前端兼容 `records/rows/list` 多种格式

### 4.2 前端最佳实践

#### 状态管理

- 使用 Pinia 管理全局状态
- 模块级状态使用独立 store
- 避免滥用全局状态

```typescript
// stores/user.ts
import { defineStore } from 'pinia';
import { ref } from 'vue';

export const useUserStore = defineStore('user', () => {
  const user = ref(null);
  
  const login = async (params) => {
    const res = await loginApi(params);
    user.value = res.data;
  };
  
  return { user, login };
});
```

#### API调用

- 统一封装 HTTP 请求
- 添加请求拦截器处理 Token
- 添加响应拦截器处理错误

```typescript
// src/api/client.ts
const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
});

http.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);
```

#### 组件设计

- 组件职责单一
- 使用 props 传递数据
- 使用 emit 触发事件
- 避免组件嵌套过深

## 六、常见问题

### 6.1 Maven依赖冲突

**问题**: 依赖版本冲突导致编译失败

**解决方案**:
- 使用 `mvn dependency:tree` 查看依赖树
- 在 `pom.xml` 中排除冲突依赖
- 统一管理依赖版本

### 6.2 前端跨域问题

**问题**: 前端请求后端API出现CORS错误

**解决方案**:
- 在后端配置跨域允许列表
- 使用Nginx反向代理
- 开发环境使用Vite代理

### 6.3 数据库连接失败

**问题**: 后端启动时无法连接数据库

**解决方案**:
- 确认MySQL服务已启动
- 检查数据库配置（url、username、password）
- 确认防火墙未阻止端口
