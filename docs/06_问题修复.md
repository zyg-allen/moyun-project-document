## 四、v4.0 问题修复详情

### 4.0 Dashboard 与举报反馈模块（v4.0）

#### 4.0.1 TableDataInfo 分页格式
**问题**：`CmsReportController`/`CmsFeedbackController` 的 `list` 方法返回 `AjaxResult`，与 RuoYi 标准 `TableDataInfo` 分页格式 `{code, msg, rows, total}` 不一致。

**修复**：返回类型改为 `TableDataInfo`，手动构造 `rspData.setRows(result.getRecords())` + `rspData.setTotal(result.getTotal())`。

**修复文件**：`CmsReportController.java` / `CmsFeedbackController.java`

#### 4.0.2 handle 方法字段篡改风险
**问题**：`handle` 方法直接 `@RequestBody PortalReport` 更新，前端可篡改 userId/status 等字段。

**修复**：改用 `LambdaUpdateWrapper` 只更新 5 个字段（status/handleResult/handler/handleTime/updateTime），增加 status 白名单校验（仅 approved/rejected）。

#### 4.0.3 实体校验缺失
**问题**：`PortalReport`/`PortalFeedback` 无校验注解。

**修复**：`reportType` 加 `@NotBlank`+`@Size(max=32)`，`description` 加 `@NotBlank`+`@Size(max=2000)`，`targetUrl` 加 `@Size(max=500)`。

### 4.1 个人中心页 mock 数据（Bug1）

**问题**：`AuthorPage.vue` 总浏览等数据为前端累加文章列表，非真实聚合值。

**修复**：改用 `growthApi.getUserStatsById(authorId)` 调用 `GET /portal/growth/stats?userId=xxx`，返回真实聚合数据。修复 `totalViews=0` 时的 NaN% 问题。

**修复文件**：`moyun-portal/src/pages/AuthorPage.vue`

### 4.2 友情链接表格字段不显示（Bug2）

**问题**：`CmsFriendLinkVO` 漏加 `@Data` 注解，Lombok 不继承，自定义字段无 getter/setter，Jackson 序列化只输出 BaseEntity 字段。

**修复**：补充 `@Data` + `@EqualsAndHashCode(callSuper = true)`。SQL 统一 `portal_friend_link.status` 为 `0/1`。

**修复文件**：`CmsFriendLinkVO.java` / `40_fix_bugs_v4.sql` / `03_portal_init.sql`

> **经验教训**：继承 `BaseEntity` 的 VO/Entity 必须显式标注 `@Data`（或 `@Getter/@Setter`），Lombok 的 `@Data` 不会继承到子类。

### 4.3 通知菜单重复（Bug3）

**问题**：后台存在 3 个通知入口（通知公告/通知中心/通知管理），前两个 Controller 和前端页面已删除但菜单数据残留，点击 404。

**修复**：
- `01_moyun_init.sql` 清理 menu_id=107 及按钮 1035-1038
- `40_fix_bugs_v4.sql` DELETE 数据库残留菜单及角色授权
- `application.yaml` xss excludes 改为 `/cms/notification`
- 保留 `cms/notification` 作为唯一入口

### 4.4 评论管理 404（Bug4）

**问题**：`sys_menu` 中评论管理菜单 `path` 不正确，权限标识 `cms:comment:audit` 与 Controller 不一致。

**修复**：`40_fix_bugs_v4.sql` 修正 path 为 `comment`，统一 perms 为 `cms:comment:edit`，补齐菜单及按钮，授权 admin 角色。

### 4.5 关注接口 500（Bug5）

**问题**：前后端接口契约不匹配，前端调用 `POST /portal/follow/{userId}`，后端无对应 `@PostMapping("/{userId}")`。

**修复**：重构 `PortalFollowController` 为语义化 RESTful 契约，新增 `follow`/`unfollow`/`selectFollowerPage`/`selectFollowingPage` 方法，事务一致性 + 幂等性。

### 4.6 PortalReport 列长不一致（P1-1）

**问题**：实体 `@Size(max=2000)`，建表 `varchar(1000)`。

**修复**：建表脚本改为 `varchar(2000)`，`40_fix_bugs_v4.sql` 新增 `ALTER TABLE` 升级语句。

### 4.7 关注状态未初始化（P1-2）

**问题**：`AuthorPage.vue` 未调用 `checkFollow`，已关注用户按钮显示错误，点击后粉丝数不同步。

**修复**：`loadAuthorData` 中已登录且非自己主页时调用 `followApi.checkFollow` 初始化 `isFollowing`。

### 4.8 通知用户搜索下拉为空（P1-3）

**问题**：`remoteUserSearch` 用 `response.rows` 取数据，应为 `response.data.records`。

**修复**：`response.rows || []` → `response.data.records || []`。

### 4.9 通知用户类型 sys 无接口（P1-4）

**问题**：表单提供"系统用户"选项，但 `/cms/user/list` 只查 portal_user 表。

**修复**：移除 sys 选项，下拉固定为"门户用户"并 disabled。

## 六、问题处理流程

### 6.1 问题发现

1. 通过测试用例发现问题
2. 用户反馈问题
3. 代码审查发现问题

### 6.2 问题分类

| 优先级 | 说明 | 处理时限 |
|--------|------|----------|
| 🔴 高 | 影响核心功能，导致系统无法正常运行 | 24小时内 |
| 🟠 中 | 影响部分功能，用户体验受影响 | 3天内 |
| 🟡 低 | 不影响功能，仅代码质量问题 | 下一版本 |

### 6.3 修复流程

```
问题发现 → 问题记录 → 分析定位 → 修复实施 → 测试验证 → 代码审查 → 合并上线
```

### 6.4 回滚方案

如果修改后出现问题：
1. **Git回滚**: 使用 `git checkout` 回滚修改的文件
2. **数据库回滚**: 如果有备份，恢复数据库
3. **配置回滚**: 检查配置文件是否正确
