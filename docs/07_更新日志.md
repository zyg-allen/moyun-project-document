## [v4.0] - 2026-06-26

### ✅ 完成

#### Dashboard 首页改造
- ✅ 后端 `SysDashboardServiceImpl` 重构 `buildTodoTasks()`：新增举报/反馈待办构建逻辑
- ✅ 待办跳转路径规范化：举报/反馈 `/cms/report|feedback`、文章审核 `/cms/article/edit?id={id}`（query 形式）
- ✅ 通知待办 routePath 从 `/system/notification` 改为 `/cms/notification`
- ✅ `businessType` 从数字编码改为枚举名（INSERT/UPDATE/DELETE/GRANT/EXPORT/IMPORT）
- ✅ `todayNewArticles` 改用 `articleMapper.countPendingArticles()` 返回真实待审核数
- ✅ `refreshCache()` 增加 ZSet 键清理（`ZSET_KEY_HOT_ARTICLES` / `ZSET_KEY_CATEGORY_VIEWS`）
- ✅ `SysDashboardController.refreshCache()` 增加 `@PreAuthorize("@ss.hasPermi('system:dashboard:refresh')")`
- ✅ 前端 `index.vue` 增加 `cacheRefreshing` loading 绑定、`loadAll` 错误提示、`goTask` 不可达告警

#### 举报反馈模块
- ✅ 新增 `CmsReportController` / `CmsFeedbackController`（后台管理）
- ✅ `list` 方法返回 `TableDataInfo`（手动构造 rows/total，兼容 RuoYi 分页格式）
- ✅ `handle` 方法改用 `LambdaUpdateWrapper` 只更新 5 个字段（status/handleResult/handler/handleTime/updateTime），防止字段篡改
- ✅ `handle` 增加 status 白名单校验（仅允许 approved/rejected）
- ✅ `PortalReport` / `PortalFeedback` 实体增加 `@NotBlank` / `@Size` 校验注解
- ✅ 前端 `report/index.vue` / `feedback/index.vue` 的 `submitHandle` 只提交 `{id, status, handleResult}`
- ✅ SQL 脚本 `38_init_report_feedback_menu.sql` 创建菜单及按钮权限

#### Bug 修复（4 项用户反馈问题）
- ✅ **Bug1**：个人中心页（`/author/{id}`）总浏览等数据为 mock
  - `AuthorPage.vue` 移除文章列表前端累加逻辑，改用 `growthApi.getUserStatsById(authorId)` 调用真实聚合接口 `/portal/growth/stats`
  - 修复 `totalViews=0` 时的 NaN% 问题
- ✅ **Bug2**：后台友情链接表格字段不显示
  - 根因：`CmsFriendLinkVO` 漏加 `@Data` 注解，导致自定义字段无 getter/setter，Jackson 序列化只输出 BaseEntity 字段
  - 修复：补充 `@Data` + `@EqualsAndHashCode(callSuper = true)`
  - SQL `40_fix_bugs_v4.sql` 统一 `portal_friend_link.status` 为 `0/1`（原 `active/inactive`）
- ✅ **Bug3**：后台通知菜单重复（通知公告 + 通知中心 + 通知管理 3 个入口）
  - 删除 `SysNoticeController` / `SysNotificationController` 及对应前端页面
  - 删除 SQL 脚本 `36_init_notification_center_menu.sql`
  - `01_moyun_init.sql` 清理 menu_id=107 及按钮 1035-1038 的 INSERT 和角色授权
  - `application.yaml` xss excludes 从 `/system/notice` 改为 `/cms/notification`
  - 保留 `cms/notification` 作为唯一通知管理入口
- ✅ **Bug4**：评论管理页面 404
  - SQL `40_fix_bugs_v4.sql` 修正菜单 path 为 `comment`、统一 perms `cms:comment:audit → cms:comment:edit`、补齐菜单及按钮、为 admin 角色授权

#### 关注/取消关注接口整合
- ✅ 重构 `PortalFollowController` 为语义化 RESTful 契约（与前端 `follow.ts` 对齐）
  - `POST /portal/follow/{userId}` 关注（幂等）
  - `DELETE /portal/follow/{userId}` 取消关注（幂等）
  - `GET /portal/follow/check/{userId}` 检查是否已关注
  - `GET /portal/follow/{userId}/followers` 粉丝列表（分页）
  - `GET /portal/follow/{userId}/following` 关注列表（分页）
- ✅ `IPortalFollowService` 新增 `follow` / `unfollow` / `selectFollowerPage` / `selectFollowingPage` 方法
- ✅ `PortalFollowServiceImpl` 实现：事务一致性（关注记录 + 统计计数 + 成长事件原子操作）、幂等性逻辑
- ✅ `PortalSecurityConfig` 放开 `GET /portal/follow/check/**` 及 followers/following 公开访问
- ✅ 保留旧接口 `toggle/{userId}` / `isFollowing/{userId}` 向后兼容

#### 三端深度复查修复（P1 问题 4 项）
- ✅ **P1-1**：`PortalReport.description` 校验 `@Size(max=2000)` 与 DB 列 `varchar(1000)` 不一致
  - 修复：`38_init_report_feedback_menu.sql` 改为 `varchar(2000)`，`40_fix_bugs_v4.sql` 新增 `ALTER TABLE` 升级语句
- ✅ **P1-2**：`AuthorPage.vue` 遗漏 `checkFollow` 调用初始化关注状态
  - 修复：`loadAuthorData` 中已登录且非自己主页时调用 `followApi.checkFollow` 设置 `isFollowing`
- ✅ **P1-3**：`notification/index.vue` 用户搜索下拉永远为空
  - 修复：`remoteUserSearch` 的 `response.rows` → `response.data.records`
- ✅ **P1-4**：`notification/index.vue` 用户类型 `sys` 无后端接口
  - 修复：移除 sys 选项，下拉固定为"门户用户"并 disabled

#### 代码清理
- ✅ 删除 `api/cms/comment.js` 中无调用方的 `addComment` / `updateComment` 死代码
- ✅ 更新 `38_init_report_feedback_menu.sql` 注释（移除对已删除菜单的引用）

#### 文章模块优化（用户/管理员发布链路完整性）
- ✅ **前台发布页标签推荐防抖**：`PublishPage.vue` 标签推荐 `watch` 每输入一字符即触发 `getRecommendTags` 请求，过于频繁
  - 新增 `scheduleTagSuggestions`（500ms 防抖 + 标题≥5 字符或已选分类才触发）
  - `onUnmounted` 清理 pending timer，避免组件卸载后回调
- ✅ **后端标签推荐 Redis 缓存**：`PortalTagController.getRecommendTags` 每次查库且忽略 title/category 参数
  - 单一缓存键 `portal:tag:recommend:top10`，TTL 10 分钟
  - 标签增/改/删时主动 `deleteObject` 清除缓存，保证新标签及时可见
- ✅ **后台发布作者信息自动填充（P0 缺陷修复）**：`CmsArticleServiceImpl.fillAdminAuthorInfo` 原为空实现，`author_id NOT NULL` 导致未选作者时插入失败
  - 前端已传 `authorId` 则尊重；否则 `SecurityUtils.getUserId()` 反查 `portal_user.user_id`
  - 已关联：使用该 `portal_user.id` 作为 `author_id`
  - 未关联：按"后台用户应在门户用户表有账户"协定自动建户（携带 sys_user 用户名/昵称/邮箱/手机/头像，role=admin）
  - `insertArticle` 加 `@Transactional`，保证建户 + 发文原子
- ✅ **后台发布 slug / category_path 维护**：与前台 `PortalArticleServiceImpl` 链路对齐
  - 新增 `fillCategoryPath`（复用 `IPortalCategoryService`，维护 `category_path` 与 `root_category_id`）
  - 新增 `fillSlug` / `generateSlugFromTitle` / `sanitizeSlug` / `ensureSlugUnique`，保证 `uk_slug` 唯一索引不冲突
  - `insertArticle` 与 `updateArticle` 均调用
- ✅ **后台 CMS 文章编辑页字段丰富化**：`moyun-admin-vue/.../cms/article/edit.vue`
  - 新增维护字段：`publishedAt`（发布时间）、`slug`（SEO 别名）、`isCategoryRecommended`（分类推荐）
  - **轮播图动态联动**：`isCarousel=true` 时 `cover` 动态必填 + 提示 1920×600 宽屏横幅；`false` 时可选 + 提示 800×450 方正缩略图
  - `rules` 改为 `computed`，`cover` 必填性随 `isCarousel` 联动；`watch(isCarousel)` 切换时重新校验封面
  - 作者字段提示"留空则自动使用当前管理员门户账户（无账户自动建户）"

#### 文章模块发布链路完整性评估
- ✅ **前台用户发布链路（完整）**：`PortalArticleController.publish` → `PortalArticleServiceImpl.publishArticle`
  - 处理 Base64 图片 → 填充作者（`PortalSecurityUtils.getUserId`）→ 维护分类路径 → 生成 slug → 状态 pending → 记录成长事件 + 字数统计
- ✅ **后台管理员发布链路（v4.0 补齐完整）**：`CmsArticleController.add` → `CmsArticleServiceImpl.insertArticle`
  - 处理图片 → 自动映射/建户作者 → 维护分类路径 → 生成 slug → 默认 published → 发布时间
  - ⚠️ 与前台差异：后台直发为 `published`（免审核），前台为 `pending`（待审核）；后台暂未记录成长事件（管理员发文不计入作者成长，符合产品语义）

#### v4.0.4 功能细节修复（审核通知 + 草稿自动保存 + 配置缓存 + 标签推荐）
- ✅ **审核结果通知作者闭环**：`CmsArticleServiceImpl.auditArticle` 审核通过/拒绝后发送站内信通知作者（BUG-002 遗留）
  - 非阻塞设计：通知失败 catch 捕获，不影响审核主流程
  - `SysNotification` 类型 system，scope=user，userType=portal
- ✅ **前台发布页草稿自动保存**：`PublishPage.vue` 启动 30 秒定时器自动调用 `saveDraft(isAuto=true)`
  - 仅当标题或内容非空时才触发，避免空页面频繁保存
  - `onUnmounted` 清理定时器
- ✅ **SysConfig 缓存加载/清空**：`SysConfigServiceImpl` 原 TODO 空实现
  - `selectConfigByKey` 先查 Redis 缓存（`sys:config:{key}`），未命中回源 DB 并回填
  - `loadingConfigCache`：全量加载 sys_config 到 Redis
  - `clearConfigCache`：删除所有 `sys:config:*` 键
- ✅ **标签推荐逻辑利用参数**：`PortalTagController.getRecommendTags` 原完全忽略 title/category 参数
  - `TagQuery` 新增 `categoryId` 字段
  - `PortalTagMapper.xml` 添加 `categoryId` 关联查询（EXISTS portal_entity_tag + portal_article）
  - title 模糊匹配标签名，category 筛选该分类下文章使用过的标签
  - 动态构建缓存键，不同查询条件独立缓存
