# 10_功能排查清单

**文档版本**: v2.0（按业务链路重构）
**创建时间**: 2026-06-26
**用途**: 按业务链路组织功能测试，从用户进入系统到完成业务目标的完整路径，每条链路串联前后端所有节点

---

## 使用说明

**链路视角**：以"用户完成一个业务目标"为主线，把跨页面、跨端、跨接口的节点串成一条链，便于端到端验证。

每条链路结构：
```
链路名（目标）
├─ 节点1：前端操作 → 接口 → 后端处理 → 副作用
├─ 节点2：...
└─ 节点N：...
```

每节点测试字段：
| 字段 | 含义 |
|------|------|
| 节点编号 | L{链路}-{序号} |
| 节点描述 | 做了什么 |
| 前端/页面 | 在哪个页面/组件操作 |
| 接口 | 方法 + 路径 |
| 后端处理 | Controller → Service → Mapper 关键逻辑 |
| 副作用/数据变更 | DB 表变更、缓存、事件、通知等 |
| 业务规则/约束 | 校验、幂等、权限等 |
| 问题描述 | 已知或测试中发现的问题 |
| 测试是否通过 | ☐ 未测试 / ✅ 通过 / ❌ 失败 / ⚠️ 部分通过 |
| 测试时间 | - |
| 关联 BUG | BUG-X |
