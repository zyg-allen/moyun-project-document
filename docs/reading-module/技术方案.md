# 墨韵·智库 — 全功能读书模块改造技术方案

> **版本**：v1.0
> **日期**：2026-06-30
> **作者**：墨韵技术团队
> **状态**：设计阶段（待评审）
> **方案类型**：方案 B（混合模式）— 保留出版物介绍能力 + 新增网络文学/小说连载能力

---

## 一、改造目标

### 1.1 现状问题
当前"读书空间"模块是**目录式展示产品**，存在以下空洞：
- 书籍详情页只有介绍（封面/简介/作者简介/ISBN），**没有正文**
- **没有章节表**，无法承载长篇小说/连载内容
- 阅读进度表 `portal_reading_progress` 已建但**前端未接入**
- 共读活动 4 张表已建但**完全未使用**（死代码）
- 缺少书架、推荐、排行榜等小说站点核心功能
- 阅读体验缺失：无字体调节、无章节导航、无续读、无夜间模式

### 1.2 改造目标
将"读书空间"升级为**全功能读书模块**，同时兼容：
1. **出版物介绍**（实体书书评/书单/金句，保持现有能力）
2. **网络文学连载**（多章节长文小说，新建章节体系）
3. **长文阅读**（单篇长文，类似"知乎长文"模式）

### 1.3 核心原则
- **不破坏现有数据**：portal_book 表平滑扩展，不推翻重建
- **复用优先**：ArticleDetailPage 的评论、分享、MarkdownRenderer 等成熟组件直接复用
- **渐进式开发**：分阶段交付，每阶段独立可上线
- **文档同步**：每个改动有 SQL 脚本记录、代码改动记录、更新日志

## 二、整体架构

### 2.1 模块全景图

```
读书模块（Reading Module）
├── 书库（Book Library）
│   ├── 出版物（type=published）— 现有能力保留
│   │   └── 书籍详情：封面/简介/作者简介/ISBN/评分/书评/书单
│   └── 网络文学（type=novel）— 新增
│       ├── 作品详情：封面/简介/标签/字数/章节数/连载状态
│       ├── 章节目录（含 VIP 标识）
│       └── 章节正文阅读器
│
├── 书架（Bookshelf）— 新增
│   ├── 想读 / 在读 / 已读
│   ├── 阅读历史（最近阅读、续读位置）
│   └── 我的书单（创建/收藏的书单）
│
├── 发现（Discover）— 新增
│   ├── 分类推荐（玄幻/都市/科幻/言情...）
│   ├── 排行榜（热门/新书/完结/字数榜）
│   ├── 限免专区
│   └── 编辑精选
│
└── 互动（Interaction）— 增强
    ├── 评论系统（复用 ArticleDetailPage 评论组件）
    ├── 金句摘录（现有，扩展为段落划线）
    ├── 点赞/收藏/分享（复用现有能力）
    └── 追更提醒（基于共读活动表改造）
```

### 3.2 新建表清单

#### 3.2.1 `portal_book_chapter`（章节表）— 核心

```sql
-- 章节 表
DROP TABLE IF EXISTS `portal_book_chapter`;
CREATE TABLE `portal_book_chapter` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
    `book_id` BIGINT NOT NULL COMMENT '所属书籍ID',
    `title` VARCHAR(500) NOT NULL COMMENT '章节标题',
    `content` LONGTEXT COMMENT '章节正文（HTML格式，上限4GB）',
    `content_markdown` TEXT COMMENT 'Markdown原始内容（上限64KB，单章足够）',
    `editor_mode` VARCHAR(20) DEFAULT 'richtext' COMMENT '编辑器模式：richtext/markdown',
    `word_count` INT DEFAULT 0 COMMENT '字数统计',
    `chapter_no` INT NOT NULL DEFAULT 0 COMMENT '章节序号（用于排序，从1开始）',
    `volume_id` BIGINT DEFAULT NULL COMMENT '所属分卷ID（可选，支持分卷管理）',
    `is_free` TINYINT(1) DEFAULT 1 COMMENT '是否免费：1=免费，0=VIP章节',
    `price` DECIMAL(10,2) DEFAULT 0.00 COMMENT '章节单价（元，VIP章节购买）',
    `is_published` TINYINT(1) DEFAULT 0 COMMENT '是否已发布：0=草稿，1=已发布',
    `publish_time` DATETIME DEFAULT NULL COMMENT '发布时间（支持定时发布）',
    `view_count` BIGINT DEFAULT 0 COMMENT '章节浏览量',
    `create_by` VARCHAR(64) DEFAULT '' COMMENT '创建者',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_by` VARCHAR(64) DEFAULT '' COMMENT '更新者',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `remark` VARCHAR(500) DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_book_chapter_no` (`book_id`, `chapter_no`),
    KEY `idx_book_id` (`book_id`),
    KEY `idx_publish_time` (`publish_time`),
    KEY `idx_is_published` (`is_published`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='书籍章节表';
```

**设计要点**：
- `content` 用 `LONGTEXT`（4GB 上限），足以承载超长章节
- `chapter_no` 唯一约束 `(book_id, chapter_no)` 防止同书章节序号冲突
- `volume_id` 预留分卷能力（如"第一部·觉醒"）
- `is_free` + `price` 支持 VIP 章节付费
- `is_published` + `publish_time` 支持草稿与定时发布
- 不做软删除（章节误删可从后台日志恢复，避免大量 DEL 字段污染）

### 3.3 现有表扩展

#### 3.3.1 `portal_book` 表扩展字段

```sql
-- 扩展 portal_book 表：支持书籍类型、字数、章节数、连载状态
ALTER TABLE `portal_book`
    ADD COLUMN `type` VARCHAR(20) DEFAULT 'published' COMMENT '书籍类型：published=出版物，novel=网络小说，longform=长文'
    AFTER `status`;

ALTER TABLE `portal_book`
    ADD COLUMN `serial_status` VARCHAR(20) DEFAULT 'completed' COMMENT '连载状态：ongoing=连载中，completed=已完结，hiatus=暂停更新'
    AFTER `type`;

ALTER TABLE `portal_book`
    ADD COLUMN `word_count` BIGINT DEFAULT 0 COMMENT '总字数（章节字数之和）'
    AFTER `serial_status`;

ALTER TABLE `portal_book`
    ADD COLUMN `chapter_count` INT DEFAULT 0 COMMENT '总章节数'
    AFTER `word_count`;

ALTER TABLE `portal_book`
    ADD COLUMN `latest_chapter_id` BIGINT DEFAULT NULL COMMENT '最新章节ID（用于追更展示）'
    AFTER `chapter_count`;

ALTER TABLE `portal_book`
    ADD COLUMN `latest_chapter_title` VARCHAR(500) DEFAULT NULL COMMENT '最新章节标题'
    AFTER `latest_chapter_id`;

ALTER TABLE `portal_book`
    ADD COLUMN `last_update_time` DATETIME DEFAULT NULL COMMENT '最后更新时间（章节发布时同步）'
    AFTER `latest_chapter_title`;

ALTER TABLE `portal_book`
    ADD COLUMN `is_finished` TINYINT(1) DEFAULT 1 COMMENT '是否完结：1=完结，0=连载中（冗余字段，便于查询）'
    AFTER `last_update_time`;

-- 为新字段建立索引
ALTER TABLE `portal_book` ADD INDEX `idx_type` (`type`);
ALTER TABLE `portal_book` ADD INDEX `idx_serial_status` (`serial_status`);
ALTER TABLE `portal_book` ADD INDEX `idx_is_finished` (`is_finished`);
ALTER TABLE `portal_book` ADD INDEX `idx_word_count` (`word_count`);
ALTER TABLE `portal_book` ADD INDEX `idx_last_update_time` (`last_update_time`);
```

**字段说明**：
- `type`：区分出版物/网文/长文，决定前端展示方式
- `serial_status`：连载状态，比 `status` 的 active/inactive 更语义化
- `word_count` / `chapter_count`：冗余字段，章节发布/删除时同步更新，避免 COUNT 查询
- `latest_chapter_id` / `latest_chapter_title`：追更展示用，避免 JOIN 查询
- `last_update_time`：用于"最近更新"排序，章节发布时同步
- `is_finished`：冗余字段，等价于 `serial_status='completed'`，便于索引

#### 5.1.1 章节阅读器页 `/reading/book/:id/chapter/:chapterId`

**核心页面**，组件名 `ChapterReaderPage.vue`，路径 `pages/reading/ChapterReaderPage.vue`。

**布局结构**：
```
┌─────────────────────────────────────────────────┐
│ 顶部栏：返回 | 书名·章节名 | 进度条 35% | 设置 │
├─────────────────────────────────────────────────┤
│                                                 │
│            章节标题（居中、大字号）              │
│                                                 │
│  正文内容（MarkdownRenderer 渲染）              │
│  - 字号可调（14/16/18/20/22px）                 │
│  - 行距可调（1.5/1.7/1.9/2.1）                  │
│  - 主题可调（默认/米黄/深棕/夜间）              │

│                                                 │
├─────────────────────────────────────────────────┤
│ 底部导航：上一章 | 目录 | 下一章 | 写评论        │
└─────────────────────────────────────────────────┘
```

**关键功能**：
1. **字号/行距/主题**：localStorage 持久化，阅读器专用主题（不污染全局主题）
2. **阅读进度条**：顶部细条，显示当前章节在全书的位置百分比
3. **续读**：进入页面时根据 `portal_reading_progress.current_chapter_id` 自动定位
4. **滚动位置记忆**：上报 `chapter_offset` 到后端，下次进入自动滚动到该位置
5. **上下章导航**：底部按钮，预加载下一章内容（prefetch）提升翻页速度
6. **目录抽屉**：右侧滑出，显示全部章节，VIP 章节带锁标，点击跳转
7. **VIP 付费墙**：章节内容到 30% 时显示付费提示卡，遮挡剩余内容
8. **分享**：复用 ArticleDetailPage 的分享菜单（6 渠道）
9. **评论**：复用评论组件，挂在章节底部
10. **阅读时长统计**：基于 `visibilitychange` 事件 + 心跳上报（每 30s 上报一次 `read_duration_ms`）

**可复用组件**（来自 ArticleDetailPage）：
- `MarkdownRenderer` — 正文渲染
- `Breadcrumb` — 面包屑
- `BackToTop` — 返回顶部
- `SiteFooter` — 页脚

## 九、风险评估与应对

### 9.1 数据风险

| 风险 | 应对 |
|---|---|
| portal_book 扩展字段影响现有数据 | 新字段均有 DEFAULT 值，存量数据自动填充，不影响现有出版物展示 |
| 章节表 content 字段过大影响性能 | LONGTEXT 字段不参与列表查询，详情查询走主键，性能可控 |
| 阅读进度表数据量增长快 | `uk_user_book` 唯一约束限制每用户每书一条，总量 = 用户数 × 书籍数，可控 |

### 9.2 性能风险

| 风险 | 应对 |
|---|---|
| 章节列表查询慢 | 加 `idx_book_id` 索引，列表只返回元信息不含正文 |
| 阅读器首屏慢 | 下一章 prefetch，章节内容 CDN 缓存（后续优化） |
| 排行榜查询慢 | `word_count` / `reading_count` / `last_update_time` 均加索引 |
| 阅读进度高频写入 | 前端节流（30s 上报一次），后端异步写入 |

### 9.3 兼容性风险

| 风险 | 应对 |
|---|---|
| 现有出版物页面受影响 | `type` 字段默认 `published`，现有书籍自动归类，前端按 type 分支展示 |
| 现有 reading.ts API 变更 | 只新增不修改，5 个现有 API 签名不变 |
| 后台菜单冲突 | 新菜单挂在"读书空间"下，不冲突 |
