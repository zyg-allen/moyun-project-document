### 关键设计决策（待第二阶段开发时遵守）

1. **进度节流**：前端 30s 上报一次，章节切换时强制上报一次，避免高频写入
2. **续读恢复**：进入章节页时，若服务端有该章节的 `chapter_offset`，自动 `scrollTo(offset)`；无则滚到顶部
3. **偏好本地优先**：阅读偏好先读 localStorage 立即生效，再异步同步服务端（多端一致）
4. **书架去重**：DB 唯一键 UK(user_id, book_id) 保证，重复收藏走 toggle 逻辑
5. **CSS 变量方案**：阅读器正文用 `--reader-font-size` / `--reader-line-height` / `--reader-theme-bg` 等变量，通过 JS 修改根元素 style 实现实时切换，无需重渲染

### 关键设计决策

1. **推荐位与书籍解耦**：`portal_book_recommend` 只存 book_id + position + 时间窗，运营上下架不动书籍数据
2. **position 枚举**：home_banner（首页轮播）/ home_hot（首页热门）/ category_top（分类顶推）/ limit_free（限免专区）/ discover_banner（发现页轮播）
3. **排行榜无独立表**：直接基于 portal_book 的 reading_count / chapter_count / word_count / last_update_time 排序，避免冗余
4. **限免 = 推荐位 + 时间窗**：position=limit_free 的推荐位 + start_time/end_time 有效期，到期自动失效（查询时过滤）
5. **发现页聚合接口**：单次请求返回 Banner + 热门排行 + 限免 + 最近更新，减少首屏请求数
