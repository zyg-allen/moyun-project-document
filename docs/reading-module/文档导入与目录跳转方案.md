# 读书模块补充方案 — 文档导入与目录跳转

> **版本**：v1.0
> **日期**：2026-06-30
> **关联文档**：[技术方案.md](./技术方案.md)
> **状态**：设计阶段（待评审）
> **核心目标**：解决"书籍数据从哪来"的关键问题，实现文档导入→自动分章→目录跳转→数据入库的自动化闭环，同时兼容面试模块。

---

## 一、问题背景

### 1.1 用户核心诉求
1. **数据来源问题**：书籍主要从其他平台下载后导入，需要自动化入库流程
2. **文档上传自动分章**：上传 docx/txt/markdown 文件后，自动识别章节并拆分入库
3. **目录跳转**：书籍要有可点击的目录，支持章节间跳转
4. **数据入库关键步骤**：要有清晰的入库流程，支持预览、编辑、确认
5. **兼容面试模块**：这套能力以后面试模块也要用

### 1.2 现状调研结论

| 模块 | 现状 | 问题 |
|---|---|---|
| 读书-书籍 | 后台手填 27 个字段，无正文 | 完全靠手工，无导入能力 |
| 读书-章节 | 不存在 | 必须新建 |
| 面试-面经 | 后台用 `el-input textarea` 6 行录入 | **与前台 MarkdownRenderer 渲染不匹配**，格式会错乱 |
| 面试-题目 | 后台 textarea 录入，一题一行 | 离散结构化数据，适合 Excel 批量导入 |
| 面试-简历模板 | 手填下载 URL | 应走文件上传 |
| 通用 | POI 5.2.5 已有，Quill/marked/FileUpload 已有 | **缺 pdfbox**（PDF 解析） |

### 1.3 方案定位
构建一个**通用文档导入框架**，支持三种业务场景：

| 场景 | 输入 | 输出 | 分章 | 适用模块 |
|---|---|---|---|---|
| **A. 书籍章节导入** | docx/txt/markdown 文件 | 多条章节记录 | ✅ 自动分章 | 读书模块 |
| **B. 长文整体导入** | docx/txt/markdown 文件 | 单条长文记录（Markdown） | ❌ 不分章，但前台有 TOC | 面经、文章 |
| **C. 结构化批量导入** | Excel/CSV | 多条结构化记录 | ❌ 字段映射 | 题目、简历模板 |

本方案重点解决 **场景 A（核心）** 和 **场景 B（兼容）**，场景 C 复用现有 ExcelUtil。

#### 3.2.1 接口定义

```java
package com.moyun.portal.service;

/**
 * 通用文档解析服务
 * 支持 txt/markdown/docx/pdf 格式，自动识别章节
 */
public interface IDocumentParseService {

    /**
     * 解析文档并自动分章
     * @param file 上传的文件
     * @param rule 分章规则（null 时用默认规则）
     * @return 解析结果（含元信息 + 章节列表）

    DocumentParseResult parseDocument(MultipartFile file, ChapterSplitRule rule);

    /**
     * 解析文档为单篇 Markdown（不分章，用于面经/长文场景）
     */
    String parseToMarkdown(MultipartFile file);
}
```

#### 3.2.3 分章规则

```java
package com.moyun.portal.domain.query;

@Data
public class ChapterSplitRule {
    /**
     * 分章模式：
     * auto = 自动识别（默认，依次尝试中文章节正则→英文章节→Markdown标题）
     * regex = 自定义正则
     * heading = 按标题级别（h1/h2/h3）
     * fixed = 按固定字数切段（每 N 字一章）
     */
    private String mode = "auto";

    /** 自定义正则（mode=regex 时生效） */
    private String regex;

    /** 标题级别（mode=heading 时生效，如 "h2"） */
    private String headingLevel;

    /** 固定字数（mode=fixed 时生效，默认 3000） */
    private Integer fixedWordCount = 3000;

    /** 是否识别分卷（如"第一部"） */
    private Boolean detectVolume = true;

    /** 最小章节字数（低于此数合并到上一章，默认 100） */
    private Integer minChapterWords = 100;
}
```

### 3.3 自动分章算法

#### 3.3.1 默认正则规则库

```java
public class ChapterRegexLibrary {
    /**
     * 中文章节标题正则（按优先级排序）
     * 匹配：第一章、第1章、第123章、第一回、第一节、第1节、第一卷、序章、楔子、引子、尾声
     */
    public static final String[] CN_CHAPTER_PATTERNS = {
        "^\\s*第[零一二三四五六七八九十百千万\\d]+[章节回卷部篇]\\s*[\\s\\S]*?$",
        "^\\s*(序章|楔子|引子|前言|序言|后记|尾声|番外)\\s*$",
        "^\\s*Chapter\\s*\\d+.*$",
        "^\\s*第[零一二三四五六七八九十百千万\\d]+卷\\s*[\\s\\S]*?$"
    };

    /** 英文章节标题正则 */
    public static final String[] EN_CHAPTER_PATTERNS = {
        "^\\s*Chapter\\s+(\\d+|[IVXLCDM]+)\\s*[:：\\.\\-—]?\\s*.*$",
        "^\\s*Part\\s+\\d+\\s*[:：\\.\\-—]?\\s*.*$",
        "^\\s*Prologue\\s*$",
        "^\\s*Epilogue\\s*$"
    };

    /** 分卷识别正则 */
    public static final String[] VOLUME_PATTERNS = {
        "^\\s*第[零一二三四五六七八九十百千万\\d]+[卷部篇]\\s*[\\s\\S]*?$",
        "^\\s*(上卷|中卷|下卷|上部|中部|下部)\\s*$"
    };
}
```

#### 3.3.2 分章流程

```
1. 文件解析为纯文本/Markdown
   ├─ txt → 直接读 UTF-8
   ├─ markdown → 直接读
   ├─ docx → POI XWPFWordExtractor 抽取（保留段落 + 标题样式）
   └─ pdf → pdfbox PDFTextStripper 抽取

2. 按行扫描，匹配分章正则
   ├─ mode=auto：依次尝试 CN_PATTERNS → EN_PATTERNS → Markdown heading
   ├─ mode=regex：用自定义正则
   ├─ mode=heading：识别 Markdown # / ## / ###
   └─ mode=fixed：按字数切段

3. 章节合并与拆分
   ├─ 识别分卷（detectVolume=true）：卷名单独标记，不合并
   ├─ 短章节合并（< minChapterWords）：合并到上一章
   └─ 无标题内容：标记为"序言"或"楔子"

4. 元信息抽取
   ├─ 标题：首个匹配的标题，或文件名去扩展名
   ├─ 作者：docx 的 core properties，pdf 的 metadata
   └─ 总字数：所有章节字数之和

5. 返回 DocumentParseResult
```

### 3.4 导入接口设计

#### 3.4.1 章节批量导入接口（场景 A）

```java
@RestController
@RequestMapping("/portal/admin/book-chapters")
public class PortalBookChapterAdminController {

    /**
     * 上传文档并解析（不入库，仅返回解析结果供预览）
     */
    @PostMapping("/parse")
    @PreAuthorize("@ss.hasPermi('portal:bookChapter:add')")
    public AjaxResult parseDocument(@RequestParam("file") MultipartFile file,
                                     @RequestParam(required = false) ChapterSplitRule rule) {
        DocumentParseResult result = documentParseService.parseDocument(file, rule);
        return AjaxResult.success(result);
    }

    /**
     * 确认导入（将解析结果批量入库）
     */
    @PostMapping("/batch-import")
    @PreAuthorize("@ss.hasPermi('portal:bookChapter:add')")
    public AjaxResult batchImport(@RequestBody ChapterImportDTO dto) {
        // dto 包含 bookId + 章节列表（用户可能已编辑过标题/内容）
        bookChapterService.batchImportChapters(dto);
        return AjaxResult.success("导入成功");
    }

    /**
     * TXT 分章预览（不依赖文件上传，直接传文本内容）
     */
    @PostMapping("/parse-text")
    @PreAuthorize("@ss.hasPermi('portal:bookChapter:add')")
    public AjaxResult parseText(@RequestBody ParseTextDTO dto) {
        DocumentParseResult result = documentParseService.parseText(dto.getContent(), dto.getRule());
        return AjaxResult.success(result);
    }
}
```

### 3.5 入库关键步骤（事务保障）

`batchImportChapters` 的事务流程：

```java
@Transactional(rollbackFor = Exception.class)
public void batchImportChapters(ChapterImportDTO dto) {
    Long bookId = dto.getBookId();

    // 1. 校验书籍存在
    PortalBook book = portalBookMapper.selectPortalBookById(bookId);
    if (book == null) throw new ServiceException("书籍不存在");

    // 2. 查询当前最大章节序号（支持追加导入）
    int startNo = bookChapterMapper.selectMaxChapterNo(bookId);
    int chapterNo = startNo;

    // 3. 批量插入章节
    List<PortalBookChapter> chapters = new ArrayList<>();
    int totalWordCount = 0;
    for (ParsedChapter pc : dto.getChapters()) {
        chapterNo++;
        PortalBookChapter chapter = new PortalBookChapter();
        chapter.setBookId(bookId);
        chapter.setChapterNo(chapterNo);
        chapter.setTitle(pc.getTitle());
        chapter.setContent(pc.getContent());
        chapter.setContentMarkdown(pc.getContent());
        chapter.setEditorMode("markdown");
        chapter.setWordCount(pc.getWordCount());
        chapter.setIsFree(true);
        chapter.setIsPublished(dto.getAutoPublish() ? 1 : 0);
        chapter.setPublishTime(dto.getAutoPublish() ? new Date() : null);
        chapters.add(chapter);
        totalWordCount += pc.getWordCount();
    }
    bookChapterMapper.batchInsert(chapters);

    // 4. 更新书籍冗余字段（字数/章节数/最新章节/最后更新时间）
    PortalBook update = new PortalBook();
    update.setId(bookId);
    update.setWordCount(book.getWordCount() + totalWordCount);
    update.setChapterCount(book.getChapterCount() + chapters.size());
    if (!chapters.isEmpty()) {
        PortalBookChapter last = chapters.get(chapters.size() - 1);
        update.setLatestChapterId(last.getId());
        update.setLatestChapterTitle(last.getTitle());
    }
    update.setLastUpdateTime(new Date());
    portalBookMapper.updatePortalBook(update);

    // 5. 记录导入日志（可选，用于审计与回滚）
    importLogService.record(bookId, chapters.size(), totalWordCount);
}
```

#### 4.2.2 `ContentToc.vue`（单文内目录浮动栏）

**路径**：`moyun-portal/src/components/ContentToc.vue`

**用于**：面经详情、文章详情（场景 B）

```
                    ┌──────────────────┐
                    │ 📑 本页目录      │
                    │ ──────────────── │
   [正文内容]       │ # 一面·技术面    │  ← 点击滚动到锚点
                    │ # 二面·项目面    │
                    │ ## 项目深挖      │
                    │ ## 技术难点      │
                    │ # 三面·HR面      │
                    │ ──────────────── │
                    │ 阅读进度 45%     │
                    └──────────────────┘
```

**实现原理**：
1. 用 `marked.lexer()` 解析 Markdown，抽取 h1-h3 标题
2. 为每个标题生成 slug 锚点（`#yi-mian-ji-shu-mian`）
3. 用 `IntersectionObserver` 监听标题元素可见性，高亮当前目录项
4. 点击目录项 `scrollIntoView({ behavior: 'smooth' })`

**集成方式**：
- 改造 `MarkdownRenderer.vue`：渲染后给 h1-h3 加 `id` 属性，emit `headings` 事件
- `ExperienceDetailPage.vue` / `ArticleDetailPage.vue` 引入 `ContentToc`，接收 headings 渲染浮动目录

### 6.2 单文内目录（ContentToc）

**MarkdownRenderer 改造**：

```vue
<!-- MarkdownRenderer.vue -->
<script setup>
import { marked } from 'marked';
import { ref, onMounted, watch } from 'vue';

const props = defineProps<{
  content: string;
  contentMarkdown?: string;
  editorMode?: string;
}>();

const emit = defineEmits<{
  (e: 'headings', headings: Heading[]): void;
}>();

interface Heading {
  level: number;      // 1/2/3
  text: string;       // 标题文本
  slug: string;       // 锚点 id
}

const html = ref('');

function render() {
  const source = props.editorMode === 'markdown' ? props.contentMarkdown : props.content;
  // 配置 marked：为标题生成 id
  const renderer = new marked.Renderer();
  const headings: Heading[] = [];
  renderer.heading = ({ text, depth }) => {
    const slug = generateSlug(text);
    headings.push({ level: depth, text, slug });
    return `<h${depth} id="${slug}">${text}</h${depth}>`;
  };
  marked.use({ renderer });
  html.value = marked.parse(source || '');
  emit('headings', headings);
}

function generateSlug(text: string): string {
  // 中文转拼音首字母 + 去特殊字符，或直接用 hash
  return 'toc-' + Math.abs(hashCode(text)).toString(36);
}

onMounted(render);
watch(() => [props.content, props.contentMarkdown], render);
</script>
```

**ContentToc 组件**：

```vue
<!-- ContentToc.vue -->
<template>
  <div class="content-toc" v-if="headings.length > 0">
    <div class="toc-title">📑 本页目录</div>
    <ul class="toc-list">
      <li
        v-for="h in headings"
        :key="h.slug"
        :class="['toc-item', `level-${h.level}`, { active: activeSlug === h.slug }]"
        @click="scrollTo(h.slug)"
      >
        {{ h.text }}
      </li>
    </ul>
  </div>
</template>

<script setup>
const props = defineProps<{ headings: Heading[] }>();

const activeSlug = ref('');

function scrollTo(slug: string) {
  const el = document.getElementById(slug);
  if (el) el.scrollIntoView({ behavior: 'smooth', block: 'start' });
}

// IntersectionObserver 监听标题可见性
onMounted(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      entries.forEach((entry) => {
        if (entry.isIntersecting) {
          activeSlug.value = entry.target.id;
        }
      });
    },
    { rootMargin: '-80px 0px -80% 0px' }
  );
  props.headings.forEach((h) => {
    const el = document.getElementById(h.slug);
    if (el) observer.observe(el);
  });
});
</script>
```

## 七、数据入库关键步骤保障

### 7.1 入库前校验

| 校验项 | 规则 |
|---|---|
| 书籍存在 | bookId 必须有效 |
| 章节序号连续 | 追加导入时从 max(chapter_no)+1 开始 |
| 标题非空 | 每章必须有标题（无标题则用"第N章"） |
| 内容非空 | 每章内容字数 > 0 |
| 字数限制 | 单章不超过 100 万字（LONGTEXT 足够） |
| 总数限制 | 单次导入不超过 500 章 |

### 7.2 入库事务

- **批量插入**：用 MyBatis 的 `<foreach>` 一次性插入，减少 DB 交互
- **事务回滚**：任何一章插入失败，全部回滚
- **冗余字段同步**：书籍的 word_count/chapter_count/latest_chapter_* 在同一事务内更新

### 7.3 入库后处理

- **导入日志**：记录 bookId / 导入章节数 / 总字数 / 操作人 / 时间
- **缓存清理**：如有书籍详情缓存，导入后失效
- **首次章节特殊处理**：如果是书籍的第一批章节，自动设置 `latest_chapter_*`

### 7.4 异常处理

| 异常场景 | 处理 |
|---|---|
| 文件格式不支持 | 返回明确错误："仅支持 txt/markdown/docx/pdf" |
| 文件过大 | 限制单文件 50MB，超限拒绝 |
| 解析失败 | 返回错误详情，不阻断后续操作 |
| docx 加密 | 检测加密后提示"请提供未加密的文档" |
| PDF 扫描件 | 检测无文本层后提示"扫描件不支持，请提供文本版 PDF" |
| 分章结果为空 | 提示"未识别到章节，请调整分章规则或检查文档格式" |

## 九、风险评估

### 9.1 解析准确性

| 风险 | 应对 |
|---|---|
| docx 标题样式识别不准 | POI 读取段落样式（Heading1/2/3），结合字号判断 |
| PDF 排版错乱 | pdfbox 按页抽取，合并换行，提示用户检查 |
| 分章正则误匹配 | 提供"自定义正则"模式，支持预览调试 |
| 章节合并过度 | `minChapterWords` 可配，默认 100 字 |

### 9.2 性能

| 风险 | 应对 |
|---|---|
| 大文件解析慢 | 限制 50MB，异步解析（后续优化） |
| 批量插入慢 | MyBatis foreach 批量插入，500 章约 1-2 秒 |
| 内存占用 | 流式解析，避免一次性加载全文 |

### 9.3 兼容性

| 风险 | 应对 |
|---|---|
| 不同来源 docx 格式差异 | 兼容 WPS/Word/Google Docs 导出的 docx |
| TXT 编码问题 | 自动检测 UTF-8/GBK/GB2312 |
| Markdown 方言 | marked 支持 GFM，兼容主流语法 |
