### 1.4 Lombok 使用规范

项目全局启用 Lombok，**优先使用注解**而非手写 getter/setter/toString。

| 场景 | 推荐写法 | 禁止写法 |
|------|---------|---------|
| 普通实体/VO/DTO/Query | `@Data` | 手写 `getXxx()/setXxx()` |
| 继承 `BaseEntity` 的类 | `@Data`（子类注解即可） | 手写 `getParams()/setParams()` |
| 日志 | `@Slf4j` | `private static final Logger log = LoggerFactory.getLogger(...)` |
| Builder 模式 | `@Builder` + `@NoArgsConstructor` + `@AllArgsConstructor` | 手写链式 setter |

**例外情况（保留手写）**:
- `UserDetails` 实现类（`LoginUser`、`PortalLoginUser`），接口方法有明确语义
- `BaseEntity.getParams()`，内部有懒加载逻辑
- 方法上有 `@Size`、`@NotBlank` 等 Bean Validation 注解，需保留 getter 以生效
- 反射工具类中 `@SuppressWarnings("unchecked" / "rawtypes")`，用于抑制泛型警告

### 1.5 OpenAPI 注解规范

**所有 Controller 和方法必须有 OpenAPI 注解**，以便 Knife4j 自动生成 API 文档。

```java
// 类级别：@Tag
@Tag(name = "门户文章", description = "门户文章的增删改查操作接口")
@RestController
@RequestMapping("/portal/article")
public class PortalArticleController extends BaseController {

    // 方法级别：@Operation + @Parameter
    @Operation(summary = "获取文章详情", description = "根据文章ID获取文章详细信息")
    @GetMapping("/{id}")
    public AjaxResult getInfo(
        @Parameter(description = "文章ID") @PathVariable Long id) {
        // ...
    }

    @Operation(summary = "分页查询文章")
    @GetMapping("/list")
    public TableDataInfo list(ArticleQuery query) {
        // ...
    }
}
```

**必须项**:
- ✅ 每个 `@RestController` 必须有 `@Tag(name = "...", description = "...")`
- ✅ 每个接口方法必须有 `@Operation(summary = "...")`
- ✅ `@PathVariable` / `@RequestParam` / `@RequestBody` 关键参数建议加 `@Parameter(description = "...")`

### 1.6 Controller 层规范

**返回格式统一**：使用 [AjaxResult](file:///workspace/moyun-project-document/moyun-server/src/main/java/com/moyun/core/base/AjaxResult.java) 和 [TableDataInfo](file:///workspace/moyun-project-document/moyun-server/src/main/java/com/moyun/core/base/TableDataInfo.java)：

| 场景 | 返回类型 | 示例 |
|------|---------|------|
| 普通操作（新增/修改/删除/单条查询） | `AjaxResult` | `return AjaxResult.success(data)` |
| 分页列表 | `TableDataInfo` | `return getDataTable(list)` |
| 文件下载 | `void`（写 `HttpServletResponse`） | 见 CommonController |

**权限注解使用**:
- 管理端接口：`@PreAuthorize("@ss.hasPermi('system:user:list')")`
- 匿名接口：`@Anonymous`（需配合 `SecurityConfig` 的放行规则）
- 角色校验：`@PreAuthorize("@ss.hasRole('admin')")`

**方法命名建议**:

| HTTP 方法 | 语义 | 推荐方法名前缀 |
|----------|------|---------------|
| `GET /xxx/list` | 分页查询列表 | `list` / `page` |
| `GET /xxx/{id}` | 单条查询详情 | `getInfo` / `get` |
| `POST /xxx` | 新增 | `add` / `create` |
| `PUT /xxx` | 修改 | `edit` / `update` |
| `DELETE /xxx/{id}` | 删除单条 | `remove` / `delete` |
| `DELETE /xxx/batch` | 批量删除 | `removeBatch` / `batchDelete` |
| `POST /xxx/{id}/action` | 特殊业务操作 | `toggleLike` / `publish` / `setTop` |

### 1.7 Service 层规范

**接口 + 实现类分离**:

```
src/main/java/com/moyun/portal/service/
├── IPortalArticleService.java      # 接口（I 前缀）
└── impl/
    └── PortalArticleServiceImpl.java  # 实现类
```

**必须 `@Service` 注解在实现类上**，接口保持纯粹的契约定义。

### 1.8 Mapper / Repository 层规范

- **必须继承 `BaseMapper<T>`**：自动获得基础 CRUD
- **禁止在 Mapper 中写复杂业务逻辑**：复杂查询放到 Service
- **禁止手写 JDBC**：统一走 MyBatis-Plus

### 1.9 实体类规范

**继承链约定**:
- 有 `createBy/createTime/updateBy/updateTime/remark/params` 字段的实体：**继承 BaseEntity**
- 树形结构实体：**继承 TreeEntity**
- 其他：独立 POJO + `@Data`

**字段注解要求**:
- 主键：`@TableId(type = IdType.AUTO)` 或对应业务策略
- 逻辑删除：`@TableLogic` + 字段 `delFlag`
- 非表字段：`@TableField(exist = false)`
- 日期时间：`@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`

**禁止事项**:
- ❌ 禁止在 Entity 中写业务计算方法
- ❌ 禁止直接返回 Entity 给前端（必须转 VO）

### 1.10 VO / DTO / Query 区分

| 类型 | 用途 | 位置 | 示例 |
|------|------|------|------|
| **VO** (View Object) | 响应给前端展示 | `domain/vo/` | `CmsArticleVO` |
| **DTO** (Data Transfer Object) | 前端请求体 / 内部传输 | `domain/dto/` | `ArticlePublishDTO` |
| **Query** | 查询条件对象 | `domain/query/` | `ArticleQuery` |

> 三者严禁混用：Controller 入参 → DTO；Service 内查询 → Query；Controller 返回 → VO。

### 1.11 异常与日志规范

**异常抛出**：优先抛出 `ServiceException(msg)` / 自定义业务异常，**禁止**吞掉异常。

**日志级别约定**:

```java
log.error("XX失败, userId={}", userId, e);   // 异常（附异常对象）
log.warn("XX条件不满足, data={}", data);      // 可恢复问题
log.info("用户{}登录成功", username);          // 业务流程记录
log.debug("queryParams={}", params);         // 调试信息（仅 dev 启用）
```

> **禁止**：`e.printStackTrace()`、`System.out.println()`、拼接字符串打日志（使用 `{}` 占位符）。

### 1.12 异步 / 定时任务规范

- 异步任务：使用 `ext/job` 模块的 `SysJob` 框架，**禁止**手写 `new Thread()`
- 定时任务：必须有幂等设计
- 异步 Manager：使用 `AsyncManager.me.execute(task)` 提交一次性异步任务

### 1.13 配置文件规范

- 所有自定义配置项：声明在 `application.yml` 的 `ruoyi:` / `minio:` 等前缀下
- 敏感信息（账号/密码/密钥）：禁止硬编码，必须走 `${}` 占位符 + 环境变量 / VM options
- 多环境区分：`application-dev.yml` / `application-prod.yml`

### 1.14 代码风格与注释规范

**文件头注释**：所有 `.java` 文件**无需写作者信息**（由 Git 追踪），但类上必须有 Javadoc 描述职责。

**方法注释**：
- public 方法：必须有 Javadoc，标注 `@param` / `@return` / `@throws` 语义
- private 方法：简短注释即可

**Git 提交信息**：采用 Conventional Commits（`feat(portal)`、`fix(cms)`、`refactor(server)`、`docs`）。

### 1.15 提交前自检清单

- [ ] 所有 Controller 有 `@Tag` / `@Operation` 注解
- [ ] 返回 `AjaxResult` 或 `TableDataInfo`，不返回裸对象
- [ ] 敏感接口带 `@PreAuthorize`，匿名接口带 `@Anonymous`
- [ ] 使用 `@Data` / `@Slf4j`，无手写 getter/setter/Logger
- [ ] `@Sensitive` 对敏感字段脱敏（手机号、邮箱、身份证）
- [ ] 无 `e.printStackTrace()`、`System.out.println()`
- [ ] 方法命名与 HTTP 语义一致（`GET` 读，`POST` 写，`PUT` 更新，`DELETE` 删除）
- [ ] 参数校验：`@Validated` + JSR-303 注解

## 2 数据库规范

### 2.1 数据库选型约束

| 项目 | 技术 | 版本约束 | 说明 |
|------|------|---------|------|
| 主数据库 | MySQL | 8.0.33+ | InnoDB 引擎，utf8mb4 字符集 |
| 开发备选 | H2 | 与 Boot 对齐 | 仅开发期本地联调可用，生产禁止 |
| 连接池 | Druid | 1.2.23 | 见 DruidConfig |
| 缓存 | Redis | 6.0+ | 与 MySQL 读写互补 |
| 文件存储 | MinIO / 本地 | - | 禁止把二进制大文件存到 MySQL |
| ORM 框架 | MyBatis-Plus | 3.5.7 | 禁止手写原生 MyBatis 配置 |

> **严格禁止**：MongoDB、PostgreSQL、Oracle 等其他数据库引入主业务链路。

### 2.2 命名规范

#### 2.2.1 表命名

**规则 1：模块前缀 + 业务名**

| 模块 | 表名前缀 | 示例 |
|------|---------|------|
| system 系统管理（RBAC、日志、配置） | `sys_` | `sys_user`、`sys_role`、`sys_menu`、`sys_oper_log` |
| portal 门户业务 | `portal_` | `portal_article`、`portal_user`、`portal_comment`、`portal_tag` |
| ext/cms CMS 管理 | 复用 `portal_` | 无独立表，通过视图/标识区分 |
| ext/job 定时任务 | `sys_job` | `sys_job`、`sys_job_log` |
| ext/flowable 工作流 | `sys_` / flowable 自动建表 | `sys_form`、`sys_deploy_form`、`ACT_*` |
| ext/generator 代码生成 | `gen_` | `gen_table`、`gen_table_column` |
| ext/file 文件管理 | `sys_file` | `sys_file` |

**规则 2：小写下划线（snake_case）**：`portal_article` ✅ / `PortalArticle` ❌

**规则 3：关联表命名**：`{主表}_{关联表}`，如 `portal_article_tag`、`portal_user_role`

#### 2.2.2 列命名

| 语义 | 推荐命名 | 类型 | 说明 |
|------|---------|------|------|
| 主键 | `id` | BIGINT | 禁止用 `xxx_id` 做主键列 |
| 外键 | `{关联表}_id` | BIGINT | `author_id`、`category_id` |
| 创建者 | `create_by` | VARCHAR(64) | 继承 BaseEntity 自动获得 |
| 创建时间 | `create_time` | DATETIME | 同上 |
| 更新者 | `update_by` | VARCHAR(64) | 同上 |
| 更新时间 | `update_time` | DATETIME | 同上 |
| 备注 | `remark` | VARCHAR(500) | 同上 |
| 逻辑删除标识 | `del_flag` | CHAR(1) | 0=正常 2=已删除 |
| 状态 | `status` | CHAR(1) | 0=正常 1=停用 |
| 布尔开关 | `is_featured`、`is_top`、`is_carousel` | TINYINT(1) | 0=false, 1=true |
| 计数字段 | `views`、`likes`、`comments`、`share_count` | INT / BIGINT | 保持自描述 |
| 时间点 | `published_at` | DATETIME | `_at` 后缀表示时间点 |

#### 2.2.3 索引命名

| 索引类型 | 命名格式 | 示例 |
|----------|---------|------|
| 主键 | `PRIMARY`（MySQL 默认） | 无需命名 |
| 普通索引 | `idx_{表名}_{列名}` | `idx_portal_article_status` |
| 唯一索引 | `uk_{表名}_{列名}` | `uk_portal_user_username` |
| 联合索引 | `idx_{表名}_{col1_col2}` | `idx_portal_article_create_time_status` |
| 全文索引（慎用） | `ft_{表名}_{列名}` | 建议走 ElasticSearch |

### 2.3 字段类型约定

| 语义 | MySQL 类型 | Java 类型 | 说明 |
|------|-----------|-----------|------|
| 主键 ID | `BIGINT` | `Long` | 分布式自增 / Snowflake |
| 小整数（状态、布尔） | `TINYINT(1)` | `Integer` / `Boolean` | 0/1 语义 |
| 普通计数 | `INT` | `Integer` | 0 ~ 2,147,483,647 |
| 大计数（浏览量等） | `BIGINT` | `Long` | 避免溢出 |
| 金额 | `DECIMAL(10,2)` 或 `BIGINT`（分） | `BigDecimal` / `Long` | 严禁 `FLOAT`/`DOUBLE` |
| 短字符串 | `VARCHAR(32~100)` | `String` | 够用即小 |
| 长字符串（标题、URL） | `VARCHAR(255~1000)` | `String` | 超 1000 建议 TEXT |
| 文本（内容） | `TEXT` / `MEDIUMTEXT` | `String` | 非索引字段 |
| 富文本 / Markdown | `MEDIUMTEXT` / `LONGTEXT` | `String` | 禁止用 VARCHAR |
| 时间点 | `DATETIME` | `LocalDateTime` | 精确到秒 |
| 日期 | `DATE` | `LocalDate` | 不需要时分秒 |
| JSON | `JSON` | `String` + FastJSON2 | 用于扩展字段 |
| 枚举 | `VARCHAR(32)` 或 `TINYINT` | `Enum` + `@EnumValue` | 字符串可读性更好 |

**非空原则**:
- 主键 `id`：`NOT NULL AUTO_INCREMENT`
- 业务关键字段：`NOT NULL` + 默认值
- 可选字段：允许 `NULL`，代码中 `Optional` 或 `Objects.nonNull()`

**默认值建议**:
- `status CHAR(1) DEFAULT '0'`
- `del_flag CHAR(1) DEFAULT '0'`
- `views INT DEFAULT 0`
- `create_time DATETIME DEFAULT CURRENT_TIMESTAMP`
- `update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`

### 2.4 表结构通用字段约定

所有**业务表**必须包含以下审计字段（通过继承 BaseEntity 自动获得）：

```sql
CREATE TABLE portal_xxx (
    id           BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键',
    -- 业务字段...
    create_by    VARCHAR(64)   DEFAULT '' COMMENT '创建者',
    create_time  DATETIME      DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by    VARCHAR(64)   DEFAULT '' COMMENT '更新者',
    update_time  DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    remark       VARCHAR(500)  DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='业务表模板';
```

**逻辑删除约定**：需要软删除的表加 `del_flag CHAR(1) DEFAULT '0'`，Entity 类加 `@TableLogic` 注解。

### 2.5 MyBatis-Plus 使用规范

**Entity 注解**:
```java
@Data
@TableName("portal_article")            // 必须：映射表名
public class PortalArticle extends BaseEntity {
    @TableId(type = IdType.AUTO)       // 必须：主键策略
    private Long id;
    @TableField(exist = false)          // 可选：非表字段
    private String authorNickname;
    @TableLogic                          // 可选：逻辑删除字段
    private String delFlag;
}
```

**Mapper 接口**：必须继承 `BaseMapper<T>`，严禁在 Mapper 接口方法上写业务逻辑。

**XML Mapper**：位置 `moyun-server/src/main/resources/mapper/{模块}/{EntityName}Mapper.xml`

ResultMap 命名：
- 标准结果映射：`{EntityName}Result`
- CMS 专用：`Cms{EntityName}Result`（extends 标准）

公共 SQL 片段：
- `queryWhere`：portal 前台查询（过滤已发布）
- `queryWhereCms`：CMS 查询（不过滤状态）

**严禁事项**:
- ❌ 禁止用 `SELECT *`，必须显式列出字段
- ❌ 禁止手写字符串拼接 SQL
- ❌ 禁止在 XML 中写业务判断逻辑
- ❌ 禁止在 XML 中硬编码 `status = 'published'` 而不给 CMS 提供独立查询

### 2.6 索引与查询优化

**必须建索引的列**：外键关联列、唯一约束列、常用过滤列、排序字段、组合查询字段

**索引原则**：
1. 联合索引遵循最左前缀原则
2. 单表建议 ≤ 8 个索引，写频繁的表更少
3. 定期检查 `EXPLAIN` 和 Druid 慢查询监控

**慢查询规避**:
- `LIKE '%keyword%'` → `LIKE 'keyword%'` 或 ElasticSearch
- 超大分页 → 游标（上次最大 id）
- `ORDER BY RAND()` → 主键随机采样或应用层随机
- N+1 查询 → MyBatis-Plus `lambdaQuery().in()` 或 JOIN

### 2.7 事务规范

**使用方式**:
```java
@Transactional(readOnly = true)              // 读多写少的查询
public List<PortalArticle> listArticles() { ... }

@Transactional(rollbackFor = Exception.class) // 多步写入
public void publishArticle(Long id) { ... }
```

**注意事项**:
- `@Transactional` 必须标注在 public 方法上
- 不要在 Controller 层加事务，必须放到 Service 层
- 同类内部调用（`this.method()`）会绕开 AOP，导致事务失效
- 长事务严禁出现，事务内部避免 HTTP 调用、锁等待

### 2.8 数据迁移与版本管理

SQL 脚本目录：`moyun-server/sql/`
- `schema.sql`：全量初始化脚本
- `init-data.sql`：初始化数据
- `v{版本}/YYYY-MM-DD-{操作}-{表名}.sql`：版本增量脚本

**禁止的 DDL（必须评审）**:
- 大表（> 100 万行）的 ALTER TABLE：MySQL 5.x 会锁表；8.x 需评估
- 变更主键 / 唯一索引 / 外键结构
- 重命名表或列
- 删除生产列 → 改为 `del_flag` 软删除后观察 1~2 个版本

### 2.9 数据库安全

- 生产库账号与应用账号分离：应用账号只有 `SELECT/INSERT/UPDATE/DELETE`
- 敏感字段加密：密码用 BCrypt，身份证/手机号用 `@Sensitive` 脱敏
- 备份策略：每日全量 + binlog 增量，备份异地保存
- 严禁在 `.sql` 文件中写入真实生产数据

### 2.10 建表自检清单

- [ ] 表名符合 `{模块}_{业务}` 规范
- [ ] 字符集 `utf8mb4`，引擎 `InnoDB`
- [ ] 有主键 `id BIGINT AUTO_INCREMENT`
- [ ] 有 `create_by/create_time/update_by/update_time/remark` 审计字段
- [ ] 必要字段加了 `NOT NULL` + 默认值
- [ ] 外键 / 过滤列 / 排序列加了索引
- [ ] 金额用 `DECIMAL` 或 `BIGINT`（分），禁止 `FLOAT/DOUBLE`
- [ ] Entity 有 `@TableName`、`@TableId`，继承 `BaseEntity`
- [ ] Mapper 继承 `BaseMapper<T>`
- [ ] 提交了增量 DDL 脚本到 `sql/` 目录

## 3 代码安全规范

### 3.1 安全架构总览

本项目基于 Spring Security 6.3.1 构建**双 SecurityFilterChain** 认证体系：

| 安全域 | 请求前缀 | 认证方式 | 入口配置 |
|--------|---------|---------|---------|
| **管理端** | `/system/*`、`/monitor/*`、`/tool/*`、`/cms/*`、`/flowable/*`、`/job/*`、`/file/*` | JWT Token (Bearer xxx) | SecurityConfig |
| **门户端** | `/portal/*` | JWT Token | PortalSecurityConfig |

**用户对象**：
- 管理端：`LoginUser`（实现 `UserDetails`）
- 门户端：`PortalLoginUser`（实现 `UserDetails`）

> 以上两个 User 对象是 Security 框架内部使用的特殊类，**禁止使用 `@Data` 替换其手写方法**。

### 3.2 认证与授权

#### 3.2.1 权限注解使用

| 注解 | 作用 |
|------|------|
| `@PreAuthorize("@ss.hasPermi('xxx:yyy:zzz')")` | 按钮级权限校验（需配置菜单权限） |
| `@PreAuthorize("@ss.hasRole('admin')")` | 角色级校验 |
| `@Anonymous` | 允许匿名访问（白名单） |

**禁止行为**:
- ❌ 禁止在生产环境用 `permitAll()` 通配放行整段路径（如 `/**`）
- ❌ 禁止把敏感接口（用户列表、订单明细）标记为 `@Anonymous`

**必须行为**:
- ✅ 所有写操作（新增/修改/删除）必须带权限注解
- ✅ 管理端 Controller 的读接口必须带权限注解
- ✅ 确需匿名的接口必须用 `@Anonymous` 显式标注，同时补充在 SecurityConfig 白名单里

#### 3.2.2 Token 管理

- 管理端：`TokenService`（基于 JJWT）
- 门户端：`PortalTokenService`（基于 JJWT）
- 存储：Redis（key：`login_tokens:{uuid}` / `portal_tokens:{uuid}`）

**Token 安全**:
- 密钥 (`jwt.secret`) 严禁硬编码，必须通过环境变量注入（`${JWT_SECRET:...}`）
- 密钥长度 ≥ 256 bits（HMAC-SHA256 要求）
- HTTP Header 传输，禁止放在 URL QueryString
- 前端：Token 存储于 `localStorage` 或 `Pinia store`，不得放 Cookie（除非 HttpOnly + Secure）

#### 3.2.3 密码安全

- 密码存储算法：**BCrypt**（`PasswordEncoder` 的实现）
- 密码字段必须：`@JsonIgnore`（响应时不返回）+ 数据库仅存 hash（长度 ≥ 100）
- 注册/登录密码：前端 `jsencrypt` RSA 加密传输，后端解密后再 BCrypt 校验
- **严禁**：任何日志打印明文密码、任何接口返回密码

### 3.3 输入校验与注入防护

#### 3.3.1 XSS（跨站脚本）攻击防护

- 全局机制：FilterConfig + `XssFilter`（如启用）
- 富文本输出必须经过 HTML 转义 / 白名单过滤
- 禁止直接拼接 HTML 字符串到响应（前端 `v-html` 必须先过滤）

#### 3.3.2 SQL 注入防护

- ✅ 必须使用 MyBatis-Plus `#{param}` 占位符（参数预编译）
- ✅ 动态查询使用 `LambdaQueryWrapper`（无字符串拼接）
- ❌ 严禁 `${param}` 字符串拼接 SQL（除非必须做动态表名，此时须加严格白名单校验）
- ❌ 严禁直接把前端参数拼接到 `ORDER BY` / `LIMIT`，必须在 Service 层做枚举映射

#### 3.3.3 命令注入 / 路径遍历防护

- 文件名必须清理 `../`、`\0`、控制字符
- 文件类型必须校验 MIME + 实际内容字节头
- 文件大小限制在前端 + 后端双重校验
- 上传目录必须与代码执行目录严格隔离

### 3.4 敏感信息与数据脱敏

`@Sensitive(desensitizedType = DesensitizedType.XXXX)` 注解用于字段级脱敏：

| 类型 | 示例 |
|------|------|
| `MOBILE_PHONE` | `138****1234` |
| `EMAIL` | `ab****@example.com` |
| `ID_CARD` | `3301**********1234` |
| `PASSWORD` | `********` |

**必须脱敏的字段**：手机号、邮箱、身份证号、银行卡号、密码、住址、真实姓名

**日志脱敏**：日志中禁止打印完整 Token、密码、手机号、身份证

### 3.5 防重放 / 防重复提交 / 限流

#### 3.5.1 `@RepeatSubmit`（防重复提交）

适用于**写操作接口**（下单、支付、发布文章、点赞）：

```java
@RepeatSubmit(interval = 5000, message = "{repeat.submit.message}")
@PostMapping
public AjaxResult add(@RequestBody ArticlePublishDTO dto) { ... }
```

#### 3.5.2 `@RateLimiter`（接口限流）

适用于高频公开接口（注册、登录、搜索、文章列表）：

```java
@RateLimiter(key = "portal_article_list", time = 60, count = 100, limitType = LimitType.IP)
@GetMapping("/list")
public TableDataInfo list() { ... }
```

| `LimitType` | 含义 | 适用场景 |
|-------------|------|---------|
| `DEFAULT` | 全局 key 限流 | 后台管理通用接口 |
| `IP` | 按客户端 IP 限流 | 门户公开接口、注册/登录 |
| `USER` | 按用户 ID 限流 | 写操作接口 |

#### 3.5.3 幂等性要求

以下接口必须实现幂等：
- 支付/订单类（防止重复扣款）
- 批量删除（重复调用不报错）
- 文件上传（重复上传不产生重复文件）

### 3.6 越权访问防护

#### 3.6.1 水平越权（用户只能操作自己的数据）

必须校验：当前登录用户是否拥有该资源的所有权。

```java
@DeleteMapping("/article/{id}")
public AjaxResult remove(@PathVariable Long id) {
    Long currentUserId = SecurityUtils.getLoginUser().getPortalUser().getId();
    PortalArticle article = portalArticleService.getById(id);
    if (!article.getAuthorId().equals(currentUserId) && !isAdmin()) {
        return AjaxResult.error(403, "无权操作");
    }
    portalArticleService.removeById(id);
    return success();
}
```

#### 3.6.2 垂直越权（普通用户不能调用管理员接口）

- 管理端接口必须带 `@PreAuthorize("@ss.hasPermi(...)")`
- 严禁在门户端 `@Anonymous` 路径下出现管理功能

#### 3.6.3 统一异常处理

所有鉴权失败 / 参数异常 / 业务异常统一走 `GlobalExceptionHandler`，返回结构化 `AjaxResult`：

```json
{ "code": 401, "msg": "未登录或 Token 已过期" }
{ "code": 403, "msg": "无权操作该资源" }
{ "code": 404, "msg": "资源不存在" }
{ "code": 500, "msg": "服务器内部错误" }
```

**严禁**在异常信息中暴露堆栈、SQL 语句、表结构、内部路径等。

### 3.7 接口日志与操作审计

#### 3.7.1 `@Log` 注解

关键写操作接口必须标注，用于记录到 `sys_oper_log` 表：

```java
@Log(title = "文章管理", businessType = BusinessType.INSERT)
@PreAuthorize("@ss.hasPermi('cms:article:add')")
@PostMapping
public AjaxResult add(@RequestBody PortalArticle article) { ... }
```

#### 3.7.2 登录日志

登录/退出行为必须落到 `sys_login_infor`，包含 IP、浏览器、操作系统、登录结果。

### 3.8 文件与资源安全

**硬性规则**:
- 允许的扩展名白名单（`jpg/png/gif/mp4/pdf/docx`），且做 MIME + 文件头双重校验
- 文件大小上限：单文件 ≤ 10MB（头像 ≤ 2MB），由 `application.yml` 配置
- 文件存储：默认 MinIO，严禁写回 `src/main/resources`
- 文件 URL：签名 URL 或带时间戳的短链

**文件下载前必须做权限校验**（只能下载自己上传的文件）。

### 3.9 前端安全（与本规范第 4 节呼应）

| 风险 | 规避方案 |
|------|---------|
| XSS | 禁止原生 `innerHTML`；`v-html` 必须先过 HTML 白名单过滤 |
| CSRF | 单页应用 + JWT 天然免疫，但登录/支付/删除前加二次确认 |
| 点击劫持 | Nginx 加 `X-Frame-Options: SAMEORIGIN` |
| 敏感数据存储 | Token 仅存 `localStorage`，不得存明文密码、身份证 |
| 第三方依赖 | 新增依赖前扫 CVE（`npm audit`） |

### 3.10 密钥与配置项清单

以下字段必须通过环境变量注入，**禁止**提交到 Git：

```yaml
ruoyi:
  jwt:
    secret: ${JWT_SECRET:请修改为您的专属秘钥}

spring:
  datasource:
    druid:
      master:
        username: ${DB_USER:root}
        password: ${DB_PASS:yourpassword}
  data:
    redis:
      password: ${REDIS_PASS:}

minio:
  access-key: ${MINIO_AK:admin}
  secret-key: ${MINIO_SK:admin123}
```

> 若发现任何真实密钥被提交到 Git，**必须立即轮换**，并对 Git 历史做 purge 处理。

### 3.11 常见漏洞与处置

| 漏洞类型 | 典型场景 | 处置方式 |
|----------|---------|---------|
| SQL 注入 | `SELECT * FROM t WHERE name = '${name}'` | 改为 `#{name}` 或用 QueryWrapper |
| 越权访问 | 删除文章时未校验 `authorId` | 查询当前用户 + 对比所有者 |
| 暴力破解 | 登录接口无限次尝试 | `@RateLimiter(limitType=IP, count=5, time=300)` |
| 敏感信息泄漏 | 响应体返回 password 字段 | `@JsonIgnore` + `@Sensitive(PASSWORD)` |
| 任意文件上传 | 上传 `.jsp` / `.exe` 文件 | 扩展名白名单 + MIME + 文件头三重校验 |
| SSRF | 后台文件上传支持任意 URL | URL 协议白名单（仅允许 http/https） |
| 并发下单 | 重复点击造成多扣费 | `@RepeatSubmit` + 数据库乐观锁 / Redis 分布式锁 |

- [ ] 所有写操作接口均有 `@PreAuthorize` 或等价 Owner 校验
- [ ] 匿名接口清单清晰，且已加 `@Anonymous` 并在 `SecurityConfig` 白名单
- [ ] 敏感字段（密码、手机、邮箱、身份证）已 `@Sensitive` 或 `@JsonIgnore`
- [ ] 数据库连接、Redis、JWT 密钥已通过环境变量注入
- [ ] 文件上传做了扩展名 + MIME + 文件头校验

- [ ] 无日志打印明文密码、Token
- [ ] 登录接口加了 `@RateLimiter(IP)` 防暴力破解
- [ ] 关键写操作加了 `@RepeatSubmit` 和 `@Log`
- [ ] `DruidConfig` 的监控账号不是默认 admin/admin

| 事件处理 | `handle` + 名词 + 动词 | `handleDeleteClick()`、`handleFormSubmit()` |

### 4.5 API 请求规范（核心）

#### 4.5.1 管理端请求封装

**入口**：`src/utils/request.js`（Axios 实例 + 请求/响应拦截器）

**API 文件约定**（每个模块一个文件，放在 `src/api/{module}/`）：

```js
// src/api/cms/article.js
import request from '@/utils/request';

export function listArticle(query) {
  return request({ url: '/cms/article/list', method: 'get', params: query });
}

export function getArticle(id) {
  return request({ url: '/cms/article/' + id, method: 'get' });
}

export function addArticle(data) {
  return request({ url: '/cms/article', method: 'post', data: data });
}

export function updateArticle(data) {
  return request({ url: '/cms/article', method: 'put', data: data });
}

export function delArticle(id) {
  return request({ url: '/cms/article/' + id, method: 'delete' });
}

export function delArticleBatch(ids) {
  return request({ url: '/cms/article/batch', method: 'delete', data: ids });
}
```

**命名规范**：`listXxx / getXxx / addXxx / updateXxx / delXxx / delXxxBatch`，与后端 Controller 方法对齐。

#### 4.5.2 门户端请求封装

**入口**：`src/api/client.ts`（统一 Axios 实例 + 响应类型）

```ts
// src/api/client.ts
import axios from 'axios';

const client = axios.create({
  baseURL: import.meta.env.VITE_API_BASE || '/api',
  timeout: 15000,
  headers: { 'Content-Type': 'application/json' }
});

// 请求拦截器：注入 Token（从 Pinia store 读取）
// 响应拦截器：统一处理 401 跳转登录、403 提示、500 友好错误

export default client;
```

#### 4.5.3 请求方法 & HTTP 语义

| 操作 | 方法 | 后端约定 | 前端示例 |
|------|------|---------|---------|
| 列表查询 | `GET /xxx/list` | 返回 `{ rows, total }` | `listArticle(params)` |
| 单条查询 | `GET /xxx/{id}` | 返回 `{ code, data, msg }` | `getArticle(id)` |
| 新增 | `POST /xxx` | body 传对象 | `addArticle(payload)` |
| 修改 | `PUT /xxx` | body 传对象（含 id） | `updateArticle(payload)` |
| 单条删除 | `DELETE /xxx/{id}` | 路径参数 | `delArticle(id)` |
| 批量删除 | `DELETE /xxx/batch` | body 传 `[1,2,3]` | `delArticleBatch(ids)` |

#### 4.5.4 Token 管理

- 管理端：Pinia `user.js` + `auth.js` 工具
- 门户端：`Pinia user store` + `localStorage`（key：`portal_token`）
- 所有请求：`Authorization: Bearer {token}`
- 401 响应：自动清理 Token + 跳登录 + 回跳原页面
- 严禁在 URL QueryString / 日志 / 页面文案中暴露 Token

### 4.6 Pinia Store 规范

**文件组织**：`src/store/index.js` + `src/store/modules/{name}.js`

**Store 命名 & 使用**:
```js
// 定义：use + 语义 + Store
export const useUserStore = defineStore('user', {
  state: () => ({ token: '', info: null }),
```

### 4.9 路由守卫与权限

#### 4.9.1 管理端（动态路由）

```js
// src/permission.js
router.beforeEach(async (to, from, next) => {
  NProgress.start();
  if (hasToken()) {
    if (!hasLoadedRoutes()) {
      // 未加载菜单时，从后端拉取动态路由
      const routes = await generateRoutes();
      router.addRoute(...routes);
      next({ ...to, replace: true }); return;
    }
    next();
  } else {
    whiteList.includes(to.path) ? next() : next(`/login?redirect=${to.fullPath}`);
  }
});
```

#### 4.9.2 门户端

```ts
// 路由 meta 中用 requiresAuth 标注需登录
```
