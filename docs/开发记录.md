## 技术问题与解决方案

### 问题 1: Java 编译错误 - 找不到 update 方法

**错误信息**:
```
java: 找不到符号 方法 updatePortalFollow(com.moyun.portal.domain.PortalFollow)
```

**问题原因**: Service 接口、实现类和 Mapper 中缺少 update 方法

**解决方案**:
- 在 `IPortalFollowService` 接口添加 `updatePortalFollow` 方法
- 在 `PortalFollowServiceImpl` 中实现该方法
- 在 `PortalFollowMapper` 接口中添加对应的 mapper 方法
- 同样的问题也出现在 `PortalBookmark` 和 `PortalLike` 模块中，采用相同方式解决

### 问题 2: Git 分支分歧

**问题现象**: 本地分支与远程分支出现分歧

**解决方案**:
```bash
git fetch --all
git reset --hard origin/main
```

### 问题 3: 代码引用错误（旧包路径）

**问题原因**: 包结构优化后，代码中仍引用旧的包路径

**示例**:
- 旧路径: `com.moyun.common.core.redis.RedisCache`
- 新路径: `com.moyun.core.config.redis.RedisCache`

**解决方案**: 批量更新所有代码中的包路径引用

### 问题 4: 前台 API 路径与后台不匹配

**问题原因**: 前台 API 路径使用 `/api/` 或 `/` 前缀，后台控制器使用 `/portal/` 前缀

**解决方案**: 将前台所有 API 路径统一前缀为 `/portal/`

### 问题 5: 注册接口返回格式不一致

**问题**: 登录接口返回 `{ token, user }`，但注册接口只返回成功消息

**解决方案**: 修改 `PortalLoginController.register()` 方法，注册成功后自动登录，返回与登录接口一致的格式

## 重要决策记录

### 决策 1: 前后台用户表独立 vs 公用

**背景**: 讨论前台用户和后台管理员是否使用同一张用户表

**方案对比**:

| 维度 | 独立表方案 | 公用表方案 |
|------|-----------|-----------|
| **数据隔离** | ✅ 完全隔离，安全性高 | ❌ 数据混合，需通过字段区分 |
| **认证体系** | ✅ 可独立配置认证策略 | ❌ 需在同一体系中处理不同用户类型 |
| **扩展性** | ✅ 前台用户可扩展更多字段 | ⚠️ 表结构会越来越复杂 |
| **维护成本** | ⚠️ 需维护两套用户体系 | ✅ 只需维护一套 |
| **业内实践** | ✅ 多数社区平台采用此方案 | 适用于小型系统 |

**最终决策**: **采用独立表方案**

**理由**:
- 数据安全隔离
- 便于后续扩展（如第三方登录、VIP体系等）
- 符合业内最佳实践

### 决策 2: Spring Security 双 FilterChain 配置

**背景**: 实现前后台独立认证

**方案**:
- 配置两个 `SecurityFilterChain` Bean
- 通过 `requestMatchers` 区分路径
- 各自使用独立的 `UserDetailsService` 和 `TokenService`

**配置要点**:
```java
// 后台安全配置
@Order(1)
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) {
    http.securityMatcher("/system/**", "/login", ...)
        // ... 后台认证配置
}

// 前台安全配置
@Order(2)
@Bean
public SecurityFilterChain portalSecurityFilterChain(HttpSecurity http) {
    http.securityMatcher("/portal/**")
        // ... 前台认证配置
}
```

### 决策 3: 第三方登录接入方案

**背景**: 规划接入微信登录、支付宝登录

**方案**: 策略模式 + 工厂模式

**设计**:
- `SocialLoginStrategy` 接口 - 定义第三方登录统一接口
- `WeChatLoginStrategy` - 微信登录实现
- `AlipayLoginStrategy` - 支付宝登录实现
- `SocialLoginFactory` - 工厂类，根据类型创建对应的策略实例

**优势**:
- 易于扩展新的第三方登录方式
- 各登录方式逻辑隔离
- 符合开闭原则
