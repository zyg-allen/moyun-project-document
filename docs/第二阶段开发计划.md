# 第二阶段开发计划 - CMS内容管理系统

## 一、开发原则

1. **模块独立架构**：CMS模块完全独立在 `/ext/cms` 包下，方便未来扩展其他应用
2. **功能链路闭环**：前台门户 ↔ CMS后台 ↔ 数据库，保证字段一致
3. **文档同步更新**：开发记录、需求记录、数据库脚本同步更新
4. **可扩展性**：考虑未来添加第三方应用（如算命、文件转换器等）的可能性

## 二、架构设计

### 后端架构
```
ext/
└── cms/
    ├── controller/       # CMS 管理控制器
    ├── domain/
    │   ├── dto/         # 数据传输对象
    │   ├── vo/          # 视图对象
    │   └── query/       # 查询对象
    ├── mapper/          # 数据访问层
    └── service/         # 业务逻辑层
        └── impl/
```

### 前端架构
```
moyun-admin-vue/src/
├── api/cms/           # CMS API
└── views/cms/         # CMS 视图
```
