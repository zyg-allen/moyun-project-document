## 三、详细问题说明

### 3.1 后台文章管理页面字段不一致（严重）

**后端CmsArticleVO字段**:
```java
private Long id;                    // ❌ 前端使用 articleId
private String authorNickname;      // ❌ 前端使用 author
private Long views;                // ❌ 前端使用 viewCount
private String excerpt;             // ❌ 前端使用 summary
```

**前端页面使用的字段**:
```vue
<el-table-column label="文章编号" prop="articleId" />    <!-- 应为 id -->
<el-table-column label="作者" prop="author" />         <!-- 应为 authorNickname -->
<el-table-column label="浏览量" prop="viewCount" />      <!-- 应为 views -->
```

**影响**: 数据无法正确展示和保存

---

### 3.2 审核状态字段不存在（严重）

**问题**: 前端使用 `auditStatus` 字段判断审核状态，但数据库中只有 `status` 字段。

**数据库status字段值**: `draft/published/archived`

**前端审核状态**:
```vue
<el-option label="待审核" value="0" />
<el-option label="已通过" value="1" />
<el-option label="已拒绝" value="2" />
```

**影响**: 审核功能无法正常工作

---

### 3.3 后台用户表单缺少字段（高）

**缺失字段**:
- `bio` - 个人简介
- `position` - 职位

**影响**: 用户信息不完整

## 五、修复建议

### 第一优先级（必须修复）

1. **修复后台文章管理页面字段命名**
   - `articleId` → `id`
   - `author` → `authorNickname`
   - `viewCount` → `views`
   - `summary` → `excerpt`

2. **修复审核状态逻辑**
   - 将 `auditStatus` 改为 `status`
