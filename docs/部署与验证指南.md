## 数据库初始化

### 1. 创建数据库

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE moyun DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 创建用户（可选）
CREATE USER 'moyun'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON moyun.* TO 'moyun'@'localhost';
FLUSH PRIVILEGES;

-- 退出 MySQL
EXIT;
```

### 2. 执行初始化脚本

脚本位置：`moyun-server/src/main/resources/sql/`

**按顺序执行以下脚本**：

```bash
# 1. 系统管理基础表
mysql -u root -p moyun < moyun-server/src/main/resources/sql/01_moyun_init.sql

# 2. 工作流与定时任务表
mysql -u root -p moyun < moyun-server/src/main/resources/sql/02_workflow_init.sql

# 3. 门户系统表
mysql -u root -p moyun < moyun-server/src/main/resources/sql/03_portal_init.sql
```

### 3. 验证数据库表

```sql
USE moyun;

-- 查看所有表
SHOW TABLES;

-- 应该看到以下表：
-- sys_* (系统管理表)
-- qrtz_* (定时任务表)
-- act_* (Flowable 工作流表)
-- portal_* (门户系统表)

-- 验证门户用户表
DESC portal_user;

-- 验证文章表
DESC portal_article;
```

## 后端服务启动

### 1. 修改配置文件

配置文件位置：`moyun-server/src/main/resources/application.yaml`

```yaml
# 数据库配置
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/moyun?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
    username: root
    password: your_password

# Redis 配置
  redis:
    host: localhost
    port: 6379
    password: # 如有密码请填写
    database: 0
```

### 2. 编译项目

```bash
cd moyun-server

# 使用 Maven 编译
mvn clean compile

# 或者跳过测试编译
mvn clean compile -DskipTests
```

### 3. 启动后端服务

```bash
# 方式一：使用 Maven 启动
mvn spring-boot:run

# 方式二：先打包再运行
mvn clean package -DskipTests
java -jar target/moyun-server-*.jar
```

### 4. 验证后端服务

- **访问地址**：http://localhost:8080
- **Swagger 文档**：http://localhost:8080/doc.html
- **健康检查**：http://localhost:8080/actuator/health

**默认后台账号**：
- 用户名：`admin`
- 密码：`admin123`

## 常见问题排查

### 1. 数据库连接失败

**症状**：后端启动报错，提示 `Communications link failure`

**解决方案**：
- 确认 MySQL 服务已启动
- 检查 `application.yaml` 中的数据库连接配置
- 确认数据库用户名和密码正确
- 确认防火墙未阻止 3306 端口

### 2. Redis 连接失败

**症状**：后端启动报错，提示 `Unable to connect to Redis`

**解决方案**：
- 确认 Redis 服务已启动
- 检查 `application.yaml` 中的 Redis 配置
- 确认 Redis 密码（如有）正确
- 确认防火墙未阻止 6379 端口

### 3. 前台 API 请求失败

**症状**：前台页面显示错误，控制台提示 404 或 500

**解决方案**：
- 确认后端服务已启动
- 检查 API 地址配置是否正确
- 确认 API 路径前缀为 `/portal/`
- 查看浏览器开发者工具 Network 面板，查看具体错误

### 4. 端口被占用

**症状**：启动时报错 `Port 8080 was already in use` 或 `Port 5173 was already in use`

**解决方案**：
```bash
# Linux/Mac: 查找占用端口的进程
lsof -i :8080

# Windows: 查找占用端口的进程
netstat -ano | findstr :8080

# 结束进程（PID 替换为实际进程ID）
kill -9 <PID>
```

### 5. 前台依赖安装失败

**症状**：`npm install` 报错

**解决方案**：
- 尝试使用淘宝镜像源：
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```
- 删除 `node_modules` 和 `package-lock.json`，重新安装
- 检查 Node.js 版本是否符合要求

### 6. 数据库表缺失

**症状**：API 请求报错，提示 `Table 'moyun.portal_xxx' doesn't exist`

**解决方案**：
- 确认已按顺序执行所有 SQL 脚本
- 确认脚本执行时没有错误
- 重新执行初始化脚本
