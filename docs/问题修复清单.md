## 二、审核状态字段修复

### 2.1 问题分析

数据库 `portal_article` 表的 `status` 字段值:
- `draft` - 草稿
- `published` - 已发布
- `archived` - 已归档

前台CMS需要增加审核状态，建议使用单独的字段或逻辑判断。

### 2.2 建议方案

**方案A**: 使用 is_featured 和 is_top 字段组合判断审核状态
- 待审核: is_featured = null && status = 'draft'
- 已通过: status = 'published'
- 已拒绝: status = 'archived'

**方案B**: 在后端VO中添加 auditStatus 字段
- 在 CmsArticleVO 中添加 `auditStatus` 字段
- 根据业务逻辑设置审核状态

### 2.3 推荐方案

建议使用**方案B**，在后端统一处理审核逻辑。

**修改文件**:
1. `CmsArticleVO.java` - 添加 auditStatus 字段
2. `CmsArticleServiceImpl.java` - 设置审核状态逻辑
3. `article/index.vue` - 使用正确的字段
