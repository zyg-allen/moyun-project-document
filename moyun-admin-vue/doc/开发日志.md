1. 文章管理 - 文章审核、上下架、推荐
2. 分类管理 - 栏目分类树
3. 审核管理 - 待审核列表、批量审核

**应用管理 (app)** - 2个子模块:
1. 班级管理 - 班级信息配置

**系统监控 (monitor)** - 7个子模块:

### 项目结构特点

#### 模块化设计

```
src/
├── api/           # API 接口层（按业务模块划分）
├── views/         # 页面视图层（按业务模块划分）
├── components/    # 公共组件层（22个组件）
├── store/         # 状态管理层（Pinia 模块化）
├── router/        # 路由配置层
├── utils/         # 工具函数层（15个工具类）
├── plugins/       # 插件层（6个插件）
└── directive/     # 自定义指令层
```

#### 代码组织

- **API 接口**: 按业务模块组织（system、community、flowable等）
- **页面视图**: 按业务模块组织，每个模块独立目录
- **公共组件**: 通用组件统一放在 components 目录
- **工具函数**: 按功能分类（request、auth、dict等）

### 开发规范总结

#### 1. 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件夹 | kebab-case | `user-management` |
| 组件文件 | PascalCase | `UserList.vue` |
| 变量/函数 | camelCase | `getUserList` |
| 常量 | UPPER_SNAKE_CASE | `MAX_COUNT` |

#### 2. 组件规范

```vue
<script setup name="ComponentName">
// 使用 script setup 语法
// name 属性用于 keep-alive
</script>
```

#### 3. API 调用规范

```javascript
// 统一定义在 src/api/ 目录下
// 按业务模块分文件
// 使用 request 封装

import request from '@/utils/request'

export function listXxx(query) {
  return request({
    url: '/module/xxx/list',
    method: 'get',
    params: query
  })
}
```

#### 4. 权限控制规范

```vue
<!-- 按钮权限 -->
<el-button v-hasPermi="['system:user:add']">新增</el-button>

<!-- 角色权限 -->
<el-button v-hasRole="['admin']">删除</el-button>
```

### 经验总结

#### 成功经验

1. **基于成熟框架**
   - RuoYi-Vue 提供了完整的基础功能
   - 减少重复开发工作
   - 经过大量项目验证

2. **模块化设计**
   - API 按业务模块组织
   - 页面按业务模块划分
   - 便于维护和扩展

3. **工作流集成**
   - Flowable 提供强大的流程引擎
   - BPMN 设计器可视化配置
   - 支持复杂的业务流程

4. **代码生成工具**
   - 一键生成前后端代码
   - 大幅提高开发效率
   - 保证代码规范性

#### 注意事项

1. **依赖版本兼容性**
   - Vue 3.4.0 与 Element Plus 2.4.3 兼容
   - bpmn-js 11.x 需要特别注意配置
   - 升级时需测试所有功能

2. **工作流配置**
   - BPMN XML 格式要求严格
   - 流程定义需要先部署再使用
   - 注意流程变量的作用域

3. **权限配置**
   - 菜单权限需要后端配合
   - 按钮权限需要在后端配置权限标识
   - 数据权限需要配置数据范围
