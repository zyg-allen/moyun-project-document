## 接口响应格式统一

### 前台统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 分页响应格式

```json
{
  "list": [],
  "total": 100,
  "page": 1,
  "pageSize": 10
}
```

### 现有后端响应格式

现有的后端使用 `AjaxResult` 和 `TableDataInfo`，需要调整为与前台一致的格式。

## 开发优先级

### P0 - 核心功能
1. 用户登录注册
2. 文章列表和详情
3. 文章发布
4. 评论功能
5. 分类和标签

### P1 - 重要功能
1. 点赞和收藏
2. 用户个人中心
3. 搜索功能
4. 通知功能

### P2 - 增强功能
1. 关注功能
2. VIP 功能
3. 钱包功能
4. 成长体系

## 实施步骤

1. **第一步：统一响应格式**
   - 创建统一的响应对象
   - 调整现有 Controller 的返回格式

2. **第二步：完善用户模块**
   - 补充用户相关接口
   - 实现用户统计

3. **第三步：完善文章模块**
   - 调整路径映射
   - 补充相关文章推荐

4. **第四步：完善互动模块**
   - 评论、点赞、收藏
   - 关注功能

5. **第五步：完善其他模块**
   - VIP、钱包、成长体系
   - 搜索、通知

## 注意事项

1. **路径映射**：前台 API 路径为 `/api/xxx`，后端需要对应调整
2. **字段命名**：优先使用前台的字段命名
3. **日期格式**：统一使用 ISO 8601 格式
4. **认证方式**：使用 Bearer Token
5. **分页参数**：统一使用 `page` 和 `pageSize`
